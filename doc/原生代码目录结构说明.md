## 🎯 各目录职责

### bridge/ - React Native 桥接
**作用**: JavaScript ↔ Kotlin 通信桥梁

- 提供 React Native 方法接口
- 处理 Promise 回调
- 调用底层服务和管理器

**文件说明**:
- `AccessibilityModule.kt`: 提供无障碍服务检查、权限管理、悬浮窗、服务管理等接口
- `AccessibilityActionModule.kt`: 提供点击、滑动、输入等操作接口，以及应用列表获取
- `AccessibilityPackage.kt`: 注册所有桥接模块到 React Native

### service/ - Android 服务
**作用**: 管理 Android 系统服务

- 无障碍服务实现
- 前台服务管理
- 后台任务执行

**文件说明**:
- `AutoGLMAccessibilityService.kt`: 无障碍服务主类，协调手势和截图处理
- `TaskExecutionService.kt`: 前台服务，显示常驻通知，保持应用运行
- `TaskExecutionHeadlessService.kt`: Headless JS 服务，执行后台 JavaScript 任务

### ui/ - UI 管理
**作用**: 管理各种 UI 组件

- 悬浮窗显示和隐藏
- 系统对话框显示
- 通知管理

**文件说明**:
- `FloatingWindowManager.kt`: 管理悬浮窗的显示、更新和隐藏
- `SystemDialogManager.kt`: 显示系统级提示窗，自动降级到 Toast
- `NotificationManager.kt`: 显示任务完成通知

### accessibility/ - 无障碍功能
**作用**: 实现无障碍服务的核心功能

- 手势操作实现
- 截图功能实现

**文件说明**:
- `GestureHandler.kt`: 实现点击、长按、双击、滑动等手势操作
- `ScreenshotHandler.kt`: 实现不同 Android 版本的截图功能

### utils/ - 工具类
**作用**: 提供通用工具方法

- 数据转换
- 资源管理
- 服务管理

**文件说明**:
- `ScreenshotManager.kt`: Bitmap 到 base64 的转换和压缩
- `WakeLockManager.kt`: WakeLock 的获取、释放和清理
- `ServiceManager.kt`: 服务的启动、更新和停止

### core/ - 核心应用
**作用**: 应用入口和配置

- 应用启动
- 模块注册
- 权限请求

**文件说明**:
- `MainActivity.kt`: 主 Activity，处理 MediaProjection 权限请求
- `MainApplication.kt`: 主 Application，注册 React Native 模块和服务
