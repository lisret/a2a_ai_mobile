# 文本输入方案梳理（非Root方案）

## 概述

本文档梳理了当前项目中所有可用的文本输入方案，所有方案均**不需要Root权限**，适用于普通Android设备。

## 方案架构

```
文本输入调用流程
├── 主入口：performTextInputAction()
│   └── 执行策略：无障碍服务优先，失败后回退到ADB
│
├── 方案1：无障碍服务输入（AccessibilityService）
│   ├── 子方案1.1：ACTION_SET_TEXT（直接设置文本）
│   └── 子方案1.2：ACTION_PASTE（剪贴板粘贴）
│
└── 方案2：ADB输入（ADBService）
    └── 使用 adb shell input text 命令
```

## 方案详细说明

### 方案1：无障碍服务输入（AccessibilityService）

**实现位置：**
- React Native桥接层：`AccessibilityActionModule.kt` (第108-121行)
- 核心实现：`AutoGLMAccessibilityService.kt` (第287-361行)
- TypeScript调用层：`AccessibilityActions.ts` (第139-167行)

**调用路径：**
```
performTextInputAction()
  → AccessibilityActionModule.performTextInput()
    → AutoGLMAccessibilityService.performTextInput()
```

**实现原理：**

#### 子方案1.1：ACTION_SET_TEXT（直接设置文本）

**适用版本：** Android 6.0 (API 23) 及以上

**实现步骤：**
1. 将文本复制到剪贴板（作为备用）
2. 获取当前窗口的根节点（`rootInActiveWindow`）
3. 查找焦点输入框节点（`findFocus(AccessibilityNodeInfo.FOCUS_INPUT)`）
4. 使用 `ACTION_SET_TEXT` 直接设置文本内容

**代码实现：**
```kotlin
// 查找焦点输入框节点
val focusedNode = rootNode.findFocus(AccessibilityNodeInfo.FOCUS_INPUT)

if (focusedNode != null && Build.VERSION.SDK_INT >= Build.VERSION_CODES.M) {
    val arguments = Bundle().apply {
        putCharSequence(
            AccessibilityNodeInfo.ACTION_ARGUMENT_SET_TEXT_CHARSEQUENCE, 
            text
        )
    }
    val success = focusedNode.performAction(
        AccessibilityNodeInfo.ACTION_SET_TEXT, 
        arguments
    )
    if (success) {
        return true  // 输入成功
    }
}
```

**优点：**
- 直接设置文本，速度快
- 不需要模拟键盘输入
- 兼容性好（Android 6.0+）

**缺点：**
- 需要输入框已获得焦点
- 部分输入框可能不支持此操作
- 需要无障碍服务权限

#### 子方案1.2：ACTION_PASTE（剪贴板粘贴）

**适用版本：** 所有Android版本

**实现步骤：**
1. 将文本复制到剪贴板
2. 获取当前窗口的根节点
3. 查找焦点输入框节点
4. 执行 `ACTION_PASTE` 粘贴操作

**代码实现：**
```kotlin
// 1. 复制到剪贴板
val clipboard = getSystemService(CLIPBOARD_SERVICE) as ClipboardManager
val clip = ClipData.newPlainText("text", text)
clipboard.setPrimaryClip(clip)

// 2. 查找焦点输入框并粘贴
val focusedNode = rootNode.findFocus(AccessibilityNodeInfo.FOCUS_INPUT)
if (focusedNode != null) {
    val pasteSuccess = focusedNode.performAction(
        AccessibilityNodeInfo.ACTION_PASTE
    )
    if (pasteSuccess) {
        return true  // 粘贴成功
    }
}
```

**优点：**
- 兼容所有Android版本
- 实现简单，可靠性较高
- 适用于大多数输入框

**缺点：**
- 需要输入框已获得焦点
- 需要输入框支持粘贴操作
- 部分安全输入框可能禁止粘贴

**当前实现策略：**
- 优先尝试 `ACTION_SET_TEXT`（Android 6.0+）
- 如果失败，回退到 `ACTION_PASTE`
- 如果都失败，返回 `false`，触发ADB回退

### 方案2：ADB输入（ADBService）

**实现位置：**
- TypeScript实现：`ADBService.ts` (第101-127行)
- 调用路径：`AccessibilityActions.ts` (第157-162行)

**调用路径：**
```
performTextInputAction()
  → executeActionWithFallback()
    → ADBService.performTextInput()
      → executeShellCommand('input text "...")
```

**实现原理：**

使用Android ADB的 `input text` 命令模拟键盘输入。

**代码实现：**
```typescript
async performTextInput(text: string): Promise<void> {
  // 转义特殊字符
  const escapedText = text
    .replace(/\\/g, '\\\\')
    .replace(/ /g, '%s')      // 空格使用 %s 转义
    .replace(/&/g, '\\&')
    .replace(/</g, '\\<')
    .replace(/>/g, '\\>')
    .replace(/\(/g, '\\(')
    .replace(/\)/g, '\\)')
    .replace(/"/g, '\\"')
    .replace(/'/g, "\\'");

  const command = `input text "${escapedText}"`;
  await this.executeShellCommand(command);
}
```

**特殊字符转义规则：**
- 空格：使用 `%s` 转义（ADB input text 标准方式）
- 反斜杠：`\\` → `\\\\`
- 其他特殊字符：`&`, `<`, `>`, `(`, `)`, `"`, `'` 都需要转义

**优点：**
- 不需要输入框已获得焦点（会自动聚焦到当前输入框）
- 兼容性好，适用于大多数场景
- 不需要无障碍服务权限（但需要ADB调试权限）

**缺点：**
- 需要启用USB调试或无线ADB
- 需要安装ADB Keyboard（某些设备）
- 特殊字符需要转义处理
- 速度可能较慢（需要逐个字符输入）

**注意事项：**
- ADB Keyboard 不会像普通键盘那样占用屏幕空间
- 要确认键盘已激活，查看屏幕底部是否显示 'ADB Keyboard {ON}'
- 或者检查输入框是否处于激活/高亮状态

## 调用流程

### 完整调用链

```
用户触发文本输入
  ↓
performTextInputAction(text, getSettingsService, getAdbService)
  ↓
executeActionWithFallback(
    '文本输入',
    async () => {
      // 尝试无障碍服务
      return await AccessibilityActionModule.performTextInput(text);
    },
    async () => {
      // ADB回退
      const adbService = await getAdbService();
      await adbService.performTextInput(text);
    },
    getSettingsService
  )
  ↓
  如果无障碍服务失败 && ADB回退已启用
    ↓
    自动切换到ADB输入
```

### 回退机制

**触发条件：**
1. 无障碍服务返回 `false`
2. 无障碍服务抛出异常
3. 用户已启用ADB回退（通过设置检查）

**回退逻辑：**
```typescript
try {
  const success = await actionFn();  // 无障碍服务
  if (!success) {
    throw new Error('操作执行失败');
  }
} catch (error) {
  // 检查ADB回退是否启用
  const adbFallbackEnabled = await settingsService.getADBFallbackEnabled();
  
  if (adbFallbackEnabled) {
    await adbFallbackFn();  // 使用ADB回退
  } else {
    throw error;  // 抛出错误
  }
}
```

## 使用建议

### 最佳实践

1. **优先使用无障碍服务**
   - 速度快，用户体验好
   - 不需要ADB连接

2. **确保输入框已获得焦点**
   - 在执行 `Type` 操作前，先执行 `Tap` 操作点击输入框
   - 检查输入框是否处于激活/高亮状态

3. **启用ADB回退**
   - 作为备用方案，提高可靠性
   - 在设置中启用 "ADB回退" 选项

4. **处理特殊字符**
   - ADB输入需要转义特殊字符
   - 无障碍服务自动处理，无需转义

### 常见问题处理

#### 问题1：无障碍服务输入失败

**可能原因：**
- 输入框未获得焦点
- 输入框不支持 `ACTION_SET_TEXT` 或 `ACTION_PASTE`
- 无障碍服务未启用

**解决方案：**
1. 在执行输入前先点击输入框
2. 检查无障碍服务是否正常运行
3. 启用ADB回退作为备用方案

#### 问题2：ADB输入失败

**可能原因：**
- ADB未连接
- ADB Keyboard未安装或未启用
- 特殊字符转义错误

**解决方案：**
1. 检查USB调试或无线ADB是否启用
2. 安装并启用ADB Keyboard
3. 检查特殊字符转义是否正确

#### 问题3：输入框不支持粘贴

**可能原因：**
- 安全输入框（如密码框）禁止粘贴
- 输入框未实现粘贴功能

**解决方案：**
1. 使用ADB输入（模拟键盘输入）
2. 或使用 `ACTION_SET_TEXT`（如果支持）

## 总结

当前项目实现了**两种非Root文本输入方案**：

1. **无障碍服务输入**（主方案）
   - 子方案1：ACTION_SET_TEXT（Android 6.0+）
   - 子方案2：ACTION_PASTE（所有版本）
   - 需要无障碍权限，速度快

2. **ADB输入**（回退方案）
   - 使用 `adb shell input text` 命令
   - 需要ADB调试权限，兼容性好

**调用策略：**
- 优先使用无障碍服务
- 失败后自动回退到ADB（如果已启用）
- 所有方案均**不需要Root权限**

**注意事项：**
- 执行输入前确保输入框已获得焦点
- 启用ADB回退以提高可靠性
- ADB输入需要处理特殊字符转义
