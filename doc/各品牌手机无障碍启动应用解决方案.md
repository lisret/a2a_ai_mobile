# 各品牌手机无障碍启动应用解决方案

## 概述

不同品牌的Android手机在无障碍服务启动应用方面存在差异，本文档总结了市面上主流手机品牌的解决方案和注意事项。

## 通用解决方案（适用于所有品牌）

### 方案1：使用 PackageManager.getLaunchIntentForPackage()（推荐）

**实现代码**：
```kotlin
val pm = context.packageManager
val intent = pm.getLaunchIntentForPackage(packageName)
intent?.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
intent?.addFlags(Intent.FLAG_ACTIVITY_CLEAR_TOP)
context.startActivity(intent)
```

**优点**：
- ✅ 不需要root权限
- ✅ 不需要系统签名
- ✅ 适用于所有Android设备
- ✅ 可以检查应用是否已安装

**缺点**：
- ⚠️ 某些品牌可能有额外的权限检查

### 方案2：使用 queryIntentActivities() 查询主Activity

**实现代码**：
```kotlin
val mainIntent = Intent(Intent.ACTION_MAIN, null)
mainIntent.addCategory(Intent.CATEGORY_LAUNCHER)
mainIntent.setPackage(packageName)

val resolveInfos = pm.queryIntentActivities(mainIntent, PackageManager.MATCH_DEFAULT_ONLY)
if (resolveInfos.isNotEmpty()) {
    val activityInfo = resolveInfos[0].activityInfo
    val activityIntent = Intent().apply {
        setClassName(packageName, activityInfo.name)
        addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
        addFlags(Intent.FLAG_ACTIVITY_CLEAR_TOP)
    }
    context.startActivity(activityIntent)
}
```

**优点**：
- ✅ 可以获取精确的Activity名称
- ✅ 适用于没有标准启动Intent的应用

### 方案3：使用 ADB 命令作为兜底方案

**实现代码**：
```bash
# 方法1: 使用 monkey 命令
monkey -p ${packageName} -c android.intent.category.LAUNCHER 1

# 方法2: 使用 am start 命令
am start -a android.intent.action.MAIN -c android.intent.category.LAUNCHER ${packageName}
```

**注意**：
- ⚠️ 某些设备可能需要特殊权限
- ⚠️ 不是所有设备都支持这些命令

## 各品牌手机详细方案

### 1. 华为（Huawei / Honor）

#### 系统特点
- **系统版本**：EMUI / HarmonyOS
- **权限要求**：相对严格，需要用户手动授权
- **后台限制**：较严格的后台应用管理

#### 解决方案

**方案A：标准Intent启动（推荐）**
```kotlin
val intent = pm.getLaunchIntentForPackage(packageName)
intent?.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
intent?.addFlags(Intent.FLAG_ACTIVITY_RESET_TASK_IF_NEEDED)
context.startActivity(intent)
```

**注意事项**：
- ✅ 需要用户手动开启无障碍服务
- ✅ 需要在"应用启动管理"中允许应用自启动
- ⚠️ HarmonyOS 3.0+ 可能需要额外的权限声明

**方案B：使用华为开放能力（可选）**
- 如果应用已集成华为HMS Core，可以使用华为提供的应用启动能力
- 需要申请华为开发者账号

#### 常见问题
1. **问题**：应用启动后立即被系统杀死
   - **解决**：在"应用启动管理"中设置为"手动管理"，允许"自启动"、"关联启动"、"后台活动"

2. **问题**：无障碍服务被系统关闭
   - **解决**：在"设置 > 电池 > 应用启动管理"中，将应用设置为"手动管理"

### 2. 小米（Xiaomi / Redmi）

#### 系统特点
- **系统版本**：MIUI
- **权限要求**：非常严格，需要多重授权
- **后台限制**：极严格的后台应用管理

#### 解决方案

**方案A：标准Intent启动 + MIUI优化**
```kotlin
val intent = pm.getLaunchIntentForPackage(packageName)
intent?.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
intent?.addFlags(Intent.FLAG_ACTIVITY_CLEAR_TOP)
// MIUI需要额外设置
intent?.putExtra("miui_start_activity", true)
context.startActivity(intent)
```

**注意事项**：
- ✅ 需要在"设置 > 应用设置 > 授权管理 > 自启动管理"中开启
- ✅ 需要在"设置 > 应用设置 > 授权管理 > 应用权限管理"中授予所有权限
- ✅ 需要在"设置 > 省电优化"中设置为"无限制"
- ✅ 需要在"设置 > 应用设置 > 应用管理 > 权限"中授予"显示在其他应用的上层"权限

**方案B：使用MIUI特定Intent标志**
```kotlin
intent?.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
intent?.addFlags(Intent.FLAG_ACTIVITY_RESET_TASK_IF_NEEDED)
// MIUI特定标志
intent?.setFlags(intent?.flags?.or(0x10000000) ?: 0)
```

#### 常见问题
1. **问题**：应用无法在后台启动其他应用
   - **解决**：在"设置 > 应用设置 > 授权管理 > 自启动管理"中开启
   - **解决**：在"设置 > 省电优化"中设置为"无限制"

2. **问题**：无障碍服务被系统自动关闭
   - **解决**：在"设置 > 应用设置 > 授权管理 > 自启动管理"中开启
   - **解决**：在"设置 > 应用设置 > 应用管理 > 权限"中授予所有权限

3. **问题**：MIUI 12+ 无法启动应用
   - **解决**：需要在"设置 > 应用设置 > 授权管理 > 应用权限管理 > 权限"中授予"显示在其他应用的上层"权限
   - **解决**：需要在"设置 > 应用设置 > 授权管理 > 自启动管理"中开启

## 通用最佳实践

### 1. 权限声明

在`AndroidManifest.xml`中声明必要的权限：

```xml
<!-- 基本权限 -->
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESSIBILITY_SERVICE" />

<!-- Android 11+ 查询所有包 -->
<uses-permission android:name="android.permission.QUERY_ALL_PACKAGES" 
    tools:ignore="QueryAllPackagesPermission" />

<!-- 显示在其他应用的上层 -->
<uses-permission android:name="android.permission.SYSTEM_ALERT_WINDOW" />
```

### 2. 启动应用的最佳实践

```kotlin
fun launchApp(packageName: String): Boolean {
    return try {
        val pm = context.packageManager
        
        // 方法1: 使用 getLaunchIntentForPackage（最常用）
        val intent = pm.getLaunchIntentForPackage(packageName)
        if (intent != null) {
            intent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
            intent.addFlags(Intent.FLAG_ACTIVITY_CLEAR_TOP)
            intent.addFlags(Intent.FLAG_ACTIVITY_RESET_TASK_IF_NEEDED)
            context.startActivity(intent)
            return true
        }
        
        // 方法2: 查询MAIN+LAUNCHER Intent
        val mainIntent = Intent(Intent.ACTION_MAIN, null)
        mainIntent.addCategory(Intent.CATEGORY_LAUNCHER)
        mainIntent.setPackage(packageName)
        
        val resolveInfos = pm.queryIntentActivities(mainIntent, PackageManager.MATCH_DEFAULT_ONLY)
        if (resolveInfos.isNotEmpty()) {
            val activityInfo = resolveInfos[0].activityInfo
            val activityIntent = Intent().apply {
                setClassName(packageName, activityInfo.name)
                addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
                addFlags(Intent.FLAG_ACTIVITY_CLEAR_TOP)
            }
            context.startActivity(activityIntent)
            return true
        }
        
        false
    } catch (e: Exception) {
        Log.e(TAG, "启动应用失败: $packageName", e)
        false
    }
}
```

### 3. 错误处理和降级方案

```kotlin
fun launchAppWithFallback(packageName: String): Boolean {
    // 方法1: 尝试无障碍服务启动
    if (launchApp(packageName)) {
        return true
    }
    
    // 方法2: 尝试ADB命令启动（需要ADB权限）
    try {
        Runtime.getRuntime().exec("am start -a android.intent.action.MAIN -c android.intent.category.LAUNCHER $packageName")
        return true
    } catch (e: Exception) {
        Log.e(TAG, "ADB启动失败", e)
    }
    
    // 方法3: 提示用户手动启动
    showManualLaunchDialog(packageName)
    return false
}
```

### 4. 品牌检测和适配

```kotlin
fun getDeviceBrand(): String {
    return Build.BRAND.lowercase()
}

fun launchAppWithBrandOptimization(packageName: String): Boolean {
    val brand = getDeviceBrand()
    val intent = pm.getLaunchIntentForPackage(packageName)
    
    when (brand) {
        "xiaomi", "redmi" -> {
            // MIUI特定优化
            intent?.putExtra("miui_start_activity", true)
        }
        "vivo" -> {
            // vivo特定优化
            intent?.putExtra("vivo_start_activity", true)
        }
        "oppo", "oneplus", "realme" -> {
            // ColorOS特定优化
            intent?.addFlags(Intent.FLAG_ACTIVITY_RESET_TASK_IF_NEEDED)
        }
    }
    
    intent?.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
    intent?.addFlags(Intent.FLAG_ACTIVITY_CLEAR_TOP)
    context.startActivity(intent)
    return true
}
```

## 总结

1. **通用方案**：使用`PackageManager.getLaunchIntentForPackage()`是最通用的方案，适用于所有品牌
2. **品牌适配**：不同品牌需要不同的权限设置和优化
3. **降级方案**：始终准备ADB命令作为降级方案
4. **用户体验**：如果自动启动失败，提供手动启动的提示
