# 打包命令参考

本文档列出 React Native Android 项目常用的打包命令。

## 重要提示

### ⚠️ 开发模式需要 Metro Bundler

在开发模式下运行应用时，**必须先启动 Metro bundler**，否则应用无法加载 JavaScript 代码。

**启动 Metro**：
```bash
# 在项目根目录执行（保持终端打开）
npm start
# 或
yarn start
```

**然后在另一个终端运行应用**：
```bash
npm run android
```

**注意**：`react-native run-android` 会自动检测并启动 Metro，但为了看到完整的日志输出，建议手动启动 Metro。

## 快速命令

### 调试版 APK（用于测试）

```bash
# Windows
cd android && gradlew.bat assembleDebug

# macOS/Linux
cd android && ./gradlew assembleDebug
```

**输出位置：** `android/app/build/outputs/apk/debug/app-debug.apk`

### 发布版 APK（用于分发）

```bash
# Windows
cd android && gradlew.bat assembleRelease

# macOS/Linux
cd android && ./gradlew assembleRelease
```

**输出位置：** `android/app/build/outputs/apk/release/app-release.apk`

### AAB 文件（用于 Google Play）

```bash
# Windows
cd android && gradlew.bat bundleRelease

# macOS/Linux
cd android && ./gradlew bundleRelease
```

**输出位置：** `android/app/build/outputs/bundle/release/app-release.aab`

## 清理和重新构建

### 清理构建缓存

```bash
# Windows
cd android && gradlew.bat clean

# macOS/Linux
cd android && ./gradlew clean
```

### 清理并重新构建

```bash
# Windows
cd android && gradlew.bat clean assembleRelease

# macOS/Linux
cd android && ./gradlew clean assembleRelease
```

## 完整打包流程

### 1. 清理项目

```bash
# 清理 node_modules（可选）
rm -rf node_modules
npm install --legacy-peer-deps

# 清理 Android 构建缓存
cd android
gradlew.bat clean  # Windows
# 或
./gradlew clean    # macOS/Linux
cd ..
```

### 2. 启动 Metro 打包器（必需，用于开发模式）

**重要**：在开发模式下，必须先启动 Metro bundler，否则应用无法加载 JavaScript 代码。

```bash
# 在项目根目录执行
npm start
# 或
yarn start
```

**注意**：
- Metro bundler 会在端口 8081 上运行
- 保持这个终端窗口打开，不要关闭
- 在另一个终端窗口执行打包命令
- 如果看到 "A dev server is already running"，说明 Metro 已经在运行

### 3. 生成 APK

```bash
# 调试版
cd android && gradlew.bat assembleDebug

# 发布版（需要配置签名）
cd android && gradlew.bat assembleRelease
```

## 使用 npm scripts（推荐）

可以在 `package.json` 中添加打包脚本：

```json
{
  "scripts": {
    "build:android:debug": "cd android && gradlew.bat assembleDebug",
    "build:android:release": "cd android && gradlew.bat assembleRelease",
    "build:android:bundle": "cd android && gradlew.bat bundleRelease",
    "clean:android": "cd android && gradlew.bat clean"
  }
}
```

然后使用：

```bash
npm run build:android:debug
npm run build:android:release
npm run build:android:bundle
npm run clean:android
```

## 查看构建信息

### 查看 APK 信息

```bash
# 使用 aapt（需要 Android SDK）
aapt dump badging app-release.apk

# 查看包名、版本号等
aapt dump badging app-release.apk | grep package
```

### 验证 APK 签名

```bash
# 使用 jarsigner（JDK 自带）
jarsigner -verify -verbose -certs app-release.apk
```

### 查看 APK 大小和架构

```bash
# 使用 aapt 查看
aapt dump badging app-release.apk | grep native-code
```

## 常见打包选项

### 只打包特定架构

在 `android/gradle.properties` 中添加：

```properties
# 只打包 arm64-v8a（推荐，覆盖大多数现代设备）
reactNativeArchitectures=arm64-v8a

# 打包多个架构
reactNativeArchitectures=armeabi-v7a,arm64-v8a
```

### 启用代码混淆

在 `android/app/build.gradle` 中：

```gradle
buildTypes {
    release {
        minifyEnabled true
        proguardFiles getDefaultProguardFile("proguard-android.txt"), "proguard-rules.pro"
    }
}
```

### 启用资源压缩

在 `android/app/build.gradle` 中：

```gradle
buildTypes {
    release {
        shrinkResources true
        minifyEnabled true
    }
}
```

## 自动化脚本

### Windows 批处理脚本 (build-release.bat)

```batch
@echo off
echo 开始清理...
cd android
call gradlew.bat clean

echo 开始构建发布版 APK...
call gradlew.bat assembleRelease

if %ERRORLEVEL% EQU 0 (
    echo.
    echo ========================================
    echo 构建成功！
    echo APK 位置: android\app\build\outputs\apk\release\app-release.apk
    echo ========================================
) else (
    echo.
    echo ========================================
    echo 构建失败！
    echo ========================================
)

cd ..
pause
```

### macOS/Linux Shell 脚本 (build-release.sh)

```bash
#!/bin/bash

echo "开始清理..."
cd android
./gradlew clean

echo "开始构建发布版 APK..."
./gradlew assembleRelease

if [ $? -eq 0 ]; then
    echo ""
    echo "========================================"
    echo "构建成功！"
    echo "APK 位置: android/app/build/outputs/apk/release/app-release.apk"
    echo "========================================"
else
    echo ""
    echo "========================================"
    echo "构建失败！"
    echo "========================================"
fi

cd ..
```

使用前需要添加执行权限：

```bash
chmod +x build-release.sh
```

## Metro Bundler 端口占用问题

### 问题：端口 8081 已被占用

如果看到错误 `EADDRINUSE: address already in use :::8081`，说明 Metro bundler 已经在运行。

### 解决方法 1: 使用现有 Metro（推荐）

如果 Metro 已经在运行，直接使用即可，不需要重新启动。

**检查 Metro 是否运行**：
```bash
# Windows
netstat -ano | findstr :8081

# macOS/Linux
lsof -i :8081
```

**如果看到输出**，说明 Metro 正在运行，可以直接运行 `npm run android`。

### 解决方法 2: 停止现有 Metro

如果需要重新启动 Metro（例如清除缓存），先停止现有进程：

**Windows**：
```bash
# 方法 1: 使用 npm 脚本（推荐，跨平台）
npm run kill:metro

# 方法 2: 使用批处理脚本
.\scripts\kill-metro.bat

# 方法 3: 使用 PowerShell 脚本
.\scripts\kill-metro.ps1

# 方法 4: 手动终止（当前进程 ID: 27776）
taskkill /F /PID 27776

# 方法 5: 终止所有 node 进程（谨慎使用）
taskkill /F /IM node.exe
```

**macOS/Linux**：
```bash
# 查找占用端口的进程
lsof -i :8081

# 终止进程
kill -9 <PID>

# 或终止所有 node 进程
killall node
```

**PowerShell**：
```powershell
# 使用脚本
.\scripts\kill-metro.ps1

# 或手动执行
Get-NetTCPConnection -LocalPort 8081 | Select-Object -ExpandProperty OwningProcess | ForEach-Object { Stop-Process -Id $_ -Force }
```

### 解决方法 3: 使用不同端口

如果不想停止现有 Metro，可以使用不同端口：

```bash
npm start -- --port 8082
```

然后在 `android/app/src/main/AndroidManifest.xml` 中配置新端口（不推荐，复杂）。

## 故障排查

### 构建失败常见原因

1. **Gradle 版本不兼容**
   ```bash
   # 查看 Gradle 版本
   cd android && gradlew.bat --version
   ```

2. **依赖下载失败**
   ```bash
   # 清理 Gradle 缓存
   cd android && gradlew.bat clean --refresh-dependencies
   ```

3. **内存不足**
   ```bash
   # 在 android/gradle.properties 中增加内存
   org.gradle.jvmargs=-Xmx4096m -XX:MaxPermSize=512m
   ```

4. **签名配置错误**
   - 检查 `android/app/build.gradle` 中的签名配置
   - 检查密钥库文件是否存在
   - 检查密码是否正确

### 查看详细构建日志

```bash
# Windows
cd android && gradlew.bat assembleRelease --info

# macOS/Linux
cd android && ./gradlew assembleRelease --info
```

### 查看堆栈跟踪

```bash
# Windows
cd android && gradlew.bat assembleRelease --stacktrace

# macOS/Linux
cd android && ./gradlew assembleRelease --stacktrace
```

## 性能优化

### 启用 Gradle 守护进程

在 `android/gradle.properties` 中：

```properties
org.gradle.daemon=true
org.gradle.parallel=true
org.gradle.caching=true
org.gradle.configureondemand=true
```

### 使用本地 Gradle 分发

在 `android/gradle/wrapper/gradle-wrapper.properties` 中指定本地 Gradle：

```properties
distributionUrl=file:///path/to/gradle-8.0-all.zip
```
