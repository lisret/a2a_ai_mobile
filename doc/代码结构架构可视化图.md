# 代码结构架构可视化图

## 🎯 核心设计理念

```
┌─────────────────────────────────────────────────────────────┐
│                    设计原则                                    │
│                                                               │
│  1. 明确职责 - 每个模块都知道自己是干什么的                    │
│  2. 模块化   - 高内聚、低耦合，易于维护                        │
│  3. 易扩展   - 新增功能无需大改现有代码                        │
│  4. 配置集中 - 所有硬编码统一管理，易于修改                    │
└─────────────────────────────────────────────────────────────┘
```

### 配置使用示例

```typescript
// ✅ 正确：从配置中导入
import { 
  TASK_CONFIG, 
  API_REQUEST_CONFIG,
  COLORS,
  STORAGE_KEYS 
} from '@shared/constants';

// 使用任务配置
const maxSteps = TASK_CONFIG.MAX_STEPS;

// 使用API配置
const temperature = API_REQUEST_CONFIG.TEMPERATURE;

// 使用UI配置
const primaryColor = COLORS.primary;

// 使用存储键
await AsyncStorage.setItem(STORAGE_KEYS.MODELS, data);

// ❌ 错误：硬编码
const maxSteps = 50;
const temperature = 0.0;
const primaryColor = '#2563EB';
await AsyncStorage.setItem('@autoglm:models', data);
```
