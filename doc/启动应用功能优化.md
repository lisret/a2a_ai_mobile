# 启动应用功能优化

## 问题描述

模型返回的是应用名称（如"浏览器"），而不是包名（如"com.android.browser"），导致启动应用失败。

## 解决方案

### 1. 智能识别输入类型

`launchApp()` 方法现在会：
- 自动判断输入是包名还是应用名称
- 包名特征：包含点号（.）且不包含空格，如 `com.example.app`
- 应用名称特征：不包含点号或包含空格，如 `浏览器`、`微信`

### 2. 包名启动（直接启动）

如果输入是包名，使用以下方法依次尝试：

1. **直接启动主Activity**：`am start -n {packageName}/.MainActivity`
2. **Intent方式启动**：`am start -a android.intent.action.MAIN -c android.intent.category.LAUNCHER {packageName}`
3. **Monkey命令启动**：`monkey -p {packageName} -c android.intent.category.LAUNCHER 1`

### 3. 应用名称启动（智能查找）

如果输入是应用名称，使用以下方法依次尝试：

1. **常见应用映射表**：通过预定义的映射表查找包名
2. **系统查询**：通过系统命令查询包名（预留接口）
3. **特定类型应用**：对于浏览器等特定类型，尝试常见包名列表

对于浏览器类应用，会尝试以下常见包名：

- org.mozilla.firefox

## 使用示例

### 示例1：使用包名启动

```typescript
await adbService.launchApp('com.tencent.mm'); // 启动微信
```

### 示例2：使用应用名称启动

```typescript
await adbService.launchApp('微信'); // 自动查找包名并启动
await adbService.launchApp('浏览器'); // 自动查找包名并启动
```

## 错误处理

如果所有方法都失败，会抛出详细的错误信息：

```
无法通过应用名称启动应用: {appName}。请使用应用包名（如：com.example.app）
```

## 后续优化建议

1. **系统查询实现**：实现 `findPackageNameBySystemQuery()` 方法，通过系统命令查询包名
2. **扩展映射表**：根据实际使用情况，扩展常见应用映射表
3. **缓存机制**：缓存应用名称到包名的映射，避免重复查询
4. **用户选择**：如果找到多个匹配的应用，让用户选择

## 注意事项

1. **权限要求**：某些启动方法可能需要系统权限或root权限
2. **性能考虑**：系统查询可能较慢，优先使用映射表
3. **兼容性**：不同设备可能安装不同的应用，映射表可能不完整
