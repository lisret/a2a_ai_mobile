# 配置整理说明

## 概述

将所有写死的配置根据功能作用整理到统一的配置文件中，便于维护和修改。

## 配置文件位置

所有配置统一存放在：`src/constants/index.ts`

## 配置分类

### 1. ADB_CONFIG - ADB服务配置

**位置**：`src/constants/index.ts`

**包含内容**：
- `COMMON_APP_PACKAGES`: 常见应用名称到包名的映射表
- `BROWSER_PACKAGES`: 浏览器类应用的常见包名列表

**使用位置**：
- `src/services/ADBService.ts`

**示例**：
```typescript
import { ADB_CONFIG } from '../constants';

// 使用应用映射表
const packageName = ADB_CONFIG.COMMON_APP_PACKAGES['微信'];

// 使用浏览器包名列表
for (const pkg of ADB_CONFIG.BROWSER_PACKAGES) {
  // ...
}
```

### 2. AUTOGLM_CONFIG - AutoGLM服务配置

**位置**：`src/constants/index.ts`

**包含内容**：
- `DEFAULT_SCREEN_SIZE`: 默认屏幕分辨率（当无法获取实际分辨率时使用）
- `ACTION_HISTORY.MAX_STEPS`: 操作历史最多包含的步数
- `API_REQUEST.TEMPERATURE`: API请求的temperature参数
- `API_REQUEST.MAX_TOKENS`: API请求的max_tokens参数
- `DEFAULT_CONFIRMATION_MESSAGE`: 默认确认消息

**使用位置**：
- `src/services/AutoGLMService.ts`

**示例**：
```typescript
import { AUTOGLM_CONFIG } from '../constants';

// 使用默认屏幕分辨率
const screenSize = AUTOGLM_CONFIG.DEFAULT_SCREEN_SIZE;

// 使用操作历史配置
const maxSteps = AUTOGLM_CONFIG.ACTION_HISTORY.MAX_STEPS;

// 使用API请求配置
const request = {
  temperature: AUTOGLM_CONFIG.API_REQUEST.TEMPERATURE,
  max_tokens: AUTOGLM_CONFIG.API_REQUEST.MAX_TOKENS,
};
```

### 3. TASK_TIMEOUT_CONFIG - 任务执行超时配置

**位置**：`src/constants/index.ts`

**包含内容**：
- `CONFIRMATION_TIMEOUT_MS`: 用户确认超时时间（30秒）
- `INTERACT_TIMEOUT_MS`: 用户选择超时时间（60秒）
- `TAKE_OVER_TIMEOUT_MS`: 用户接管超时时间（5分钟）
- `ERROR_RETRY_DELAY_MS`: 错误后重试延迟时间（1秒）
- `UI_UPDATE_DELAY_MS`: 界面更新等待时间（500毫秒）

**使用位置**：
- `src/services/TaskExecutionHeadless.ts`

**示例**：
```typescript
import { TASK_TIMEOUT_CONFIG } from '../constants';

// 使用确认超时
setTimeout(() => {
  resolve(false);
}, TASK_TIMEOUT_CONFIG.CONFIRMATION_TIMEOUT_MS);

// 使用错误重试延迟
await new Promise(resolve => 
  setTimeout(resolve, TASK_TIMEOUT_CONFIG.ERROR_RETRY_DELAY_MS)
);
```

### 4. PROMPT_CONFIG - 提示词配置

**位置**：`src/constants/index.ts`

**包含内容**：
- `WEEKDAY_NAMES`: 星期名称数组（中文）

**使用位置**：
- `src/services/prompts_zh.ts`

**示例**：
```typescript
import { PROMPT_CONFIG } from '../constants';

const weekday = PROMPT_CONFIG.WEEKDAY_NAMES[today.getDay()];
```

### 5. TASK_CONFIG - 任务执行配置（已存在）

**位置**：`src/constants/index.ts`

**包含内容**：
- `MAX_STEPS`: 最大步骤数
- `MAX_CONSECUTIVE_ERRORS`: 最大连续错误数
- `API_TIMEOUT_MS`: API调用超时时间
- `SCREENSHOT_TIMEOUT_MS`: 截图超时时间
- `ACTION_WAIT_DURATION`: 操作后等待时间
- `BACKGROUND_DELAY_MS`: 切换到后台后的延迟时间
- `MAX_TASKS`: 最大任务记录数

**使用位置**：
- `src/services/TaskExecutionHeadless.ts`
- `src/hooks/useTaskExecution.ts`

### 6. ACCESSIBILITY_CONFIG - 无障碍服务配置（已存在）

**位置**：`src/constants/index.ts`

**包含内容**：
- `MAX_RETRIES`: 最大重试次数
- `RETRY_DELAY_MS`: 重试延迟时间
- `SERVICE_STARTUP_TIMEOUT_MS`: 服务启动超时时间
- `ANDROID_9_CHECK_RETRIES`: Android 9检查重试次数
- `ANDROID_9_CHECK_DELAY_MS`: Android 9检查延迟时间

## 配置修改指南

### 添加新应用映射

在 `ADB_CONFIG.COMMON_APP_PACKAGES` 中添加：

```typescript
export const ADB_CONFIG = {
  COMMON_APP_PACKAGES: {
    // ... 现有映射
    '新应用名称': 'com.example.newapp',
    'newapp': 'com.example.newapp',
  } as const,
  // ...
};
```

### 修改超时时间

在对应的配置对象中修改：

```typescript
export const TASK_TIMEOUT_CONFIG = {
  CONFIRMATION_TIMEOUT_MS: 60000, // 改为60秒
  // ...
};
```

### 修改API参数

在 `AUTOGLM_CONFIG.API_REQUEST` 中修改：

```typescript
export const AUTOGLM_CONFIG = {
  API_REQUEST: {
    TEMPERATURE: 0.5, // 降低temperature
    MAX_TOKENS: 2000, // 增加max_tokens
  } as const,
  // ...
};
```

## 优势

1. **集中管理**：所有配置在一个文件中，便于查找和修改
2. **类型安全**：使用 TypeScript 的 `as const` 确保类型安全
3. **易于维护**：修改配置不需要在多个文件中查找
4. **功能分类**：按功能分类，结构清晰
5. **避免硬编码**：减少代码中的魔法数字和字符串

## 注意事项

1. 修改配置后需要重新构建应用
2. 某些配置可能需要重启应用才能生效
3. 配置值应该根据实际使用情况调整
4. 建议在修改前备份原配置
