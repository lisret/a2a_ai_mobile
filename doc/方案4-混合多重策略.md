# 方案4：混合多重策略（最可靠方案）

## 方案概述
结合多种方法，按优先级依次尝试，确保在不同设备、不同系统版本上都能获取到最完整的应用列表。这是最可靠的方案，适用于所有场景。

## 优势
- ✅ 兼容性最好，适用于所有 Android 版本和厂商定制系统
- ✅ 可靠性最高，一种方法失败会自动尝试其他方法
- ✅ 可以获取最全面的应用列表
- ✅ 自动选择最佳方法

## 实现代码

### Kotlin 原生模块实现

```kotlin
@ReactMethod
fun getAllInstalledAppsHybrid(promise: Promise) {
    try {
        Log.d("AccessibilityActionModule", "开始使用混合策略获取所有应用")
        
        val pm = reactApplicationContext.packageManager
        val appsMap = Arguments.createMap()
        val packageSet = mutableSetOf<String>()
        var methodUsed = ""
        
        // 策略1: 优先使用 getInstalledPackages（最标准的方法）
        try {
            Log.d("AccessibilityActionModule", "尝试策略1: getInstalledPackages")
            val installedPackages = pm.getInstalledPackages(
                PackageManager.GET_META_DATA or 
                PackageManager.GET_ACTIVITIES
            )
            
            for (packageInfo in installedPackages) {
                try {
                    val packageName = packageInfo.packageName
                    val appInfo = packageInfo.applicationInfo
                    
                    val appName = try {
                        pm.getApplicationLabel(appInfo).toString()
                    } catch (e: Exception) {
                        packageName
                    }
                    
                    val launchIntent = pm.getLaunchIntentForPackage(packageName)
                    
                    if (!packageSet.contains(packageName)) {
                        if (launchIntent != null) {
                            appsMap.putString(appName, packageName)
                        } else {
                            appsMap.putString(packageName, packageName)
                        }
                        packageSet.add(packageName)
                    }
                } catch (e: Exception) {
                    // 忽略单个应用
                }
            }
            
            if (packageSet.isNotEmpty()) {
                methodUsed = "getInstalledPackages"
                Log.d("AccessibilityActionModule", "策略1成功，获取到 ${packageSet.size} 个应用")
            }
        } catch (e: Exception) {
            Log.w("AccessibilityActionModule", "策略1失败", e)
        }
        
        // 策略2: 如果策略1获取的应用较少，尝试 getInstalledApplications
        if (packageSet.size < 50) { // 如果应用数量明显偏少，尝试补充
            try {
                Log.d("AccessibilityActionModule", "尝试策略2: getInstalledApplications 补充")
                val installedApps = pm.getInstalledApplications(PackageManager.GET_META_DATA)
                
                var addedCount = 0
                for (appInfo in installedApps) {
                    try {
                        val packageName = appInfo.packageName
                        
                        if (packageSet.contains(packageName)) {
                            continue
                        }
                        
                        val appName = try {
                            pm.getApplicationLabel(appInfo).toString()
                        } catch (e: Exception) {
                            packageName
                        }
                        
                        val launchIntent = pm.getLaunchIntentForPackage(packageName)
                        
                        if (launchIntent != null) {
                            appsMap.putString(appName, packageName)
                        } else {
                            appsMap.putString(packageName, packageName)
                        }
                        packageSet.add(packageName)
                        addedCount++
                    } catch (e: Exception) {
                        // 忽略单个应用
                    }
                }
                
                if (addedCount > 0) {
                    methodUsed += " + getInstalledApplications"
                    Log.d("AccessibilityActionModule", "策略2补充了 $addedCount 个应用")
                }
            } catch (e: Exception) {
                Log.w("AccessibilityActionModule", "策略2失败", e)
            }
        }
        
        // 策略3: 通过 Intent 查询补充可启动应用
        try {
            Log.d("AccessibilityActionModule", "尝试策略3: Intent查询补充")
            val mainIntent = Intent(Intent.ACTION_MAIN, null)
            mainIntent.addCategory(Intent.CATEGORY_LAUNCHER)
            
            val resolveInfos = pm.queryIntentActivities(mainIntent, 0)
            var addedCount = 0
            
            for (resolveInfo in resolveInfos) {
                try {
                    val packageName = resolveInfo.activityInfo.packageName
                    
                    if (packageSet.contains(packageName)) {
                        continue
                    }
                    
                    val appName = try {
                        val appInfo = pm.getApplicationInfo(packageName, 0)
                        pm.getApplicationLabel(appInfo).toString()
                    } catch (e: Exception) {
                        packageName
                    }
                    
                    appsMap.putString(appName, packageName)
                    packageSet.add(packageName)
                    addedCount++
                } catch (e: Exception) {
                    // 忽略单个应用
                }
            }
            
            if (addedCount > 0) {
                methodUsed += " + Intent查询"
                Log.d("AccessibilityActionModule", "策略3补充了 $addedCount 个应用")
            }
        } catch (e: Exception) {
            Log.w("AccessibilityActionModule", "策略3失败", e)
        }
        
        // 策略4: 如果前面的方法都失败或获取的应用太少，尝试使用 MATCH_UNINSTALLED_PACKAGES
        if (packageSet.isEmpty() || packageSet.size < 10) {
            try {
                Log.d("AccessibilityActionModule", "尝试策略4: MATCH_UNINSTALLED_PACKAGES")
                val installedPackages = pm.getInstalledPackages(
                    PackageManager.GET_META_DATA or
                    PackageManager.MATCH_UNINSTALLED_PACKAGES
                )
                
                var addedCount = 0
                for (packageInfo in installedPackages) {
                    try {
                        val packageName = packageInfo.packageName
                        
                        if (packageSet.contains(packageName)) {
                            continue
                        }
                        
                        val appName = try {
                            pm.getApplicationLabel(packageInfo.applicationInfo).toString()
                        } catch (e: Exception) {
                            packageName
                        }
                        
                        val launchIntent = pm.getLaunchIntentForPackage(packageName)
                        
                        if (launchIntent != null) {
                            appsMap.putString(appName, packageName)
                        } else {
                            appsMap.putString(packageName, packageName)
                        }
                        packageSet.add(packageName)
                        addedCount++
                    } catch (e: Exception) {
                        // 忽略单个应用
                    }
                }
                
                if (addedCount > 0) {
                    methodUsed += " + MATCH_UNINSTALLED"
                    Log.d("AccessibilityActionModule", "策略4补充了 $addedCount 个应用")
                }
            } catch (e: Exception) {
                Log.w("AccessibilityActionModule", "策略4失败", e)
            }
        }
        
        Log.d("AccessibilityActionModule", "混合策略最终获取到 ${packageSet.size} 个应用，使用方法: $methodUsed")
        
        if (packageSet.isEmpty()) {
            promise.reject("ERROR", "所有策略都失败，无法获取应用列表")
        } else {
            promise.resolve(appsMap)
        }
        
    } catch (e: Exception) {
        Log.e("AccessibilityActionModule", "混合策略获取应用列表失败", e)
        promise.reject("ERROR", "获取应用列表失败: ${e.message}", e)
    }
}
```

## 策略优先级说明

### 策略1: getInstalledPackages（最标准）
- 适用于：所有 Android 版本
- 优点：标准API，信息最全
- 缺点：某些厂商可能限制

### 策略2: getInstalledApplications（补充）
- 适用于：策略1获取不完整时
- 优点：可以获取到策略1遗漏的应用
- 缺点：可能获取到一些系统组件

### 策略3: Intent查询（补充可启动应用）
- 适用于：需要确保获取所有可启动应用
- 优点：可以获取到所有有启动入口的应用
- 缺点：可能遗漏纯服务类应用

### 策略4: MATCH_UNINSTALLED_PACKAGES（兜底）
- 适用于：前面策略都失败或获取很少时
- 优点：可以获取到已卸载但保留数据的应用
- 缺点：可能包含一些不应该显示的应用

## 适用场景
- ✅ 需要最高可靠性的生产环境
- ✅ 需要兼容所有 Android 版本和厂商定制系统
- ✅ 不确定目标设备的具体情况
- ✅ 需要获取最全面的应用列表

## 注意事项
1. **性能考虑**: 多个策略会依次执行，可能影响性能
2. **结果合并**: 需要正确处理去重，避免重复应用
3. **错误处理**: 每个策略独立处理错误，不影响其他策略
4. **日志记录**: 记录使用的策略，便于调试和优化

## 优化建议
1. **缓存结果**: 应用列表变化不频繁，可以缓存较长时间
2. **条件执行**: 如果策略1成功获取足够应用，可以跳过其他策略
3. **异步处理**: 多个策略可以并行执行，提高速度
4. **厂商适配**: 针对特定厂商（如华为、小米）可以优先使用特定策略
