# 应用后台切换问题修复

## 问题描述

当任务触发后，app转为后台模式后，大概0.5秒又打开了。

## 问题分析

### 执行流程

1. **前台执行流程**（`useTaskExecutionWithBackground.ts`）：
   - 第75行：启动前台服务（在前台）
   - 第81行：启动后台任务（Headless JS）
   - 第84行：等待500ms
   - 第87行：切换到后台

2. **Headless JS 执行流程**（`TaskExecutionHeadless.ts`）：
   - 第197行：**再次启动前台服务**（在后台！）❌
   - 第221行：切换到后台

### 根本原因

**在 Headless JS 中重复启动前台服务导致应用回到前台**

- `startTaskExecutionService` 调用 `startForegroundService()`
- 在 Android 11+ (API 30+)，从后台启动前台服务可能会导致：
  - 应用回到前台（某些设备）
  - 系统限制异常（某些设备）
  - 用户体验问题（应用闪烁）

### 代码位置

**问题代码**（`TaskExecutionHeadless.ts` 第197行）：
```typescript
// ❌ 错误：在后台再次启动前台服务
await accessibilityService.startTaskExecutionService(`任务执行中...\n步骤: 0/${maxSteps}`);
```

## 修复方案

### 方案：使用 `updateTaskExecutionService` 而不是 `startTaskExecutionService`

**修复后的代码**（`TaskExecutionHeadless.ts` 第195-207行）：
```typescript
// ✅ 正确：只更新前台服务通知，不重新启动服务
try {
  // 更新前台服务通知（不重新启动服务，避免应用回到前台）
  // 如果服务未运行，updateTaskExecutionService 会启动它，但不会导致应用回到前台
  await accessibilityService.updateTaskExecutionService(`任务执行中...\n步骤: 0/${maxSteps}`);
} catch (e) {
  console.warn('[Headless JS] 更新前台服务失败（可能服务未启动）:', e);
  // 如果更新失败，尝试启动服务（作为降级方案）
  try {
    await accessibilityService.startTaskExecutionService(`任务执行中...\n步骤: 0/${maxSteps}`);
  } catch (startError) {
    console.warn('[Headless JS] 启动前台服务也失败:', startError);
  }
}
```

### 修复原理

1. **`updateTaskExecutionService` vs `startTaskExecutionService`**：
   - `startTaskExecutionService` → `startForegroundService()` → 可能导致应用回到前台
   - `updateTaskExecutionService` → `startService()` → 只更新通知，不会导致应用回到前台

2. **服务状态检查**：
   - `TaskExecutionService.onStartCommand` 中已有逻辑检查服务是否已运行
   - 如果服务已运行，只更新通知，不重新启动前台服务
   - 如果服务未运行，会启动前台服务，但使用 `startService()` 而不是 `startForegroundService()`

3. **降级方案**：
   - 如果 `updateTaskExecutionService` 失败（服务未运行），才尝试 `startTaskExecutionService`
   - 这确保了即使服务未运行，也能启动服务（虽然可能导致应用回到前台，但这是降级方案）

## 执行流程（修复后）

### 前台执行流程（不变）
```
1. 启动前台服务（在前台）✅
2. 获取 WakeLock ✅
3. 启动后台任务（Headless JS）✅
4. 等待500ms ✅
5. 切换到后台 ✅
```

### Headless JS 执行流程（修复后）
```
1. 更新前台服务通知（不重新启动服务）✅
   - 如果服务已运行：只更新通知 ✅
   - 如果服务未运行：启动服务（降级方案）⚠️
2. 获取 WakeLock ✅
3. 切换到后台 ✅
```

## 验证方法

1. **测试步骤**：
   - 启动任务
   - 观察应用是否在切换到后台后立即回到前台
   - 检查日志，确认使用的是 `updateTaskExecutionService` 而不是 `startTaskExecutionService`

2. **预期结果**：
   - 应用切换到后台后，不会立即回到前台
   - 日志中应该看到 "更新前台服务通知" 而不是 "启动前台服务"

3. **日志检查**：
   ```
   [Headless JS] 更新前台服务失败（可能服务未启动）: ...
   [TaskExecutionService] 前台服务通知已更新
   ```

## 注意事项

1. **服务状态**：
   - 确保前台服务在前台就已经启动（已在 `useTaskExecutionWithBackground.ts` 中实现）
   - Headless JS 中只更新通知，不重新启动服务

2. **降级方案**：
   - 如果服务未运行，会尝试启动服务（可能导致应用回到前台）
   - 这是降级方案，正常情况下不应该触发

3. **Android 版本差异**：
   - Android 11+ 对后台启动前台服务有更严格的限制
   - 使用 `updateTaskExecutionService` 可以避免这些限制
