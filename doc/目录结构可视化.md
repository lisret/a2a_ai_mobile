## 🏗️ 模块架构说明

### 🎯 abilityModule (能力模块)

**职责**: 提供设备操作能力（截图、点击、滑动等）

```
abilityModule/
├── code/              → 能力的具体实现
│   ├── Accessibility/ → 无障碍服务能力
│   └── ADB/          → ADB服务能力
│
├── types/             → 能力接口和类型定义
│   └── index.ts      → ICapability, CapabilityMetadata 等
│
├── services/          → 能力管理服务
│   ├── CapabilityManager.ts   → 能力注册和管理
│   └── CapabilityService.ts   → 统一的能力访问接口
│
└── index.ts          → 模块统一导出入口
```

**使用方式**:
```typescript
import { capabilityService, initializeCapabilities } from '@/abilityModule';

// 初始化
await initializeCapabilities();
await capabilityService.initialize();

// 使用能力
await capabilityService.captureScreen();
await capabilityService.performClick(100, 200);
```

### ⚙️ functionalModule (方法模块)

**职责**: 提供任务执行逻辑和功能模块

```
functionalModule/
├── code/              → 功能模块的具体实现
│   └── modules/       → 各种功能模块
│       ├── ScreenshotModule.ts        → 截图功能
│       ├── ModelInferenceModule.ts    → 模型推理功能
│       ├── ActionExecutionModule.ts   → 操作执行功能
│       └── ...                        → 其他功能模块
│
├── types/             → 方法模块类型定义
│   └── index.ts
│
├── services/          → 任务执行服务
│   ├── TaskExecutionEngine.ts  → 任务执行引擎（整合所有模块）
│   ├── TaskExecutionActions.ts → 操作执行
│   ├── TaskExecutionHelpers.ts → 辅助函数
│   └── services/               → 子服务（解析器等）
│
└── index.ts          → 模块统一导出入口
```

**使用方式**:
```typescript
import { TaskExecutionEngine } from '@/functionalModule';

const engine = new TaskExecutionEngine();
const task = await engine.execute({
  taskId: 'task_123',
  modelId: 'model_456',
  instruction: '打开微信',
  model: modelConfig,
});
```

```
└─────────────────────────────────────────────────┘
                        │
        ┌───────────────┴───────────────┐
        │                               │
        ▼                               ▼
┌──────────────────┐          ┌──────────────────┐
│  abilityModule   │          │ functionalModule │
│  (能力模块)       │          │  (方法模块)      │
│                  │          │                  │
│  - 设备操作能力   │◄─────────┤ - 任务执行逻辑   │
│  - 截图、点击等   │  使用    │ - 功能模块组合   │
└──────────────────┘          └──────────────────┘
```

## 📌 设计原则

1. **模块化**: 能力模块和方法模块完全独立，职责清晰
2. **分层结构**: 每个模块包含 `code/`、`types/`、`services/`、`index.ts` 四个标准部分
3. **统一入口**: 通过 `index.ts` 统一导出，外部只需导入模块入口
4. **可扩展性**: 新增能力或功能只需在对应模块的 `code/` 目录下添加实现
5. **类型安全**: 所有类型定义集中在 `types/` 目录，便于维护

## 🎯 目录命名规范

- **abilityModule**: 能力模块，提供设备操作能力
- **functionalModule**: 方法模块，提供任务执行功能
- **code/**: 存放具体实现代码
- **types/**: 存放类型定义文件
- **services/**: 存放服务层代码
- **index.ts**: 模块统一导出入口
