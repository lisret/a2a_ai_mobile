# 部署脚本说明

## 📦 项目打包和部署

### 方法 1: 使用 Git (推荐)

如果项目在 Git 仓库中：

```bash
# 在新电脑上克隆项目
git clone <项目仓库地址>
cd Open-AutoGLM-ReactNative/AwesomeProject

# 安装依赖
npm install

# 配置 Android SDK 路径
# 编辑 android/local.properties
```

### 方法 2: 手动复制项目

#### Windows

```powershell
# 1. 复制整个项目文件夹到新位置
# 使用文件管理器或 robocopy

# 2. 在新位置打开 PowerShell
cd <新项目路径>\AwesomeProject

# 3. 安装依赖
npm install

# 4. 配置 Android SDK
# 编辑 android\local.properties
```

#### macOS / Linux

```bash
# 1. 复制整个项目文件夹
cp -r /原路径/Open-AutoGLM-ReactNative /新路径/

# 2. 进入项目目录
cd /新路径/Open-AutoGLM-ReactNative/AwesomeProject

# 3. 安装依赖
npm install

# 4. 配置 Android SDK
# 编辑 android/local.properties
```

## 🔧 必须配置的文件

### 1. android/local.properties

**Windows**:
```properties
sdk.dir=C:\\Users\\<你的用户名>\\AppData\\Local\\Android\\Sdk
```

**macOS**:
```properties
sdk.dir=/Users/<你的用户名>/Library/Android/sdk
```

**Linux**:
```properties
sdk.dir=/home/<你的用户名>/Android/Sdk
```

### 2. 环境变量

确保以下环境变量已配置：

- `ANDROID_HOME` - Android SDK 路径
- `JAVA_HOME` - Java JDK 路径
- `PATH` - 包含 `$ANDROID_HOME/platform-tools`

## 📋 部署检查清单

在新环境中部署时，按以下顺序检查：

- [ ] **环境检查**
  - [ ] Node.js >= 18 已安装
  - [ ] Java JDK 17 已安装
  - [ ] Android Studio 已安装
  - [ ] Android SDK 已安装

- [ ] **项目文件**
  - [ ] 项目文件夹已复制
  - [ ] `package.json` 存在
  - [ ] `android/` 目录存在
  - [ ] `src/` 目录存在

- [ ] **配置文件**
  - [ ] `android/local.properties` 已创建并配置
  - [ ] 环境变量已配置

- [ ] **依赖安装**
  - [ ] `npm install` 执行成功
  - [ ] `node_modules/` 目录存在

- [ ] **验证**
  - [ ] `npm start` 可以启动 Metro
  - [ ] `npm run android` 可以构建应用

## 🚀 快速验证脚本

创建以下脚本文件来快速验证环境：

### Windows (verify-env.bat)

```batch
@echo off
echo 检查 Node.js...
node --version
if %errorlevel% neq 0 (
    echo [错误] Node.js 未安装
    exit /b 1
)

echo 检查 Java...
java -version
if %errorlevel% neq 0 (
    echo [错误] Java 未安装
    exit /b 1
)

echo 检查 Android SDK...
if not defined ANDROID_HOME (
    echo [错误] ANDROID_HOME 未设置
    exit /b 1
) else (
    echo ANDROID_HOME=%ANDROID_HOME%
)

echo 检查项目文件...
if not exist "package.json" (
    echo [错误] package.json 不存在
    exit /b 1
)

if not exist "android\local.properties" (
    echo [警告] android\local.properties 不存在，需要手动创建
)

echo.
echo [成功] 环境检查完成！
pause
```

### macOS / Linux (verify-env.sh)

```bash
#!/bin/bash

echo "检查 Node.js..."
if ! command -v node &> /dev/null; then
    echo "[错误] Node.js 未安装"
    exit 1
fi
node --version

echo "检查 Java..."
if ! command -v java &> /dev/null; then
    echo "[错误] Java 未安装"
    exit 1
fi
java -version

echo "检查 Android SDK..."
if [ -z "$ANDROID_HOME" ]; then
    echo "[错误] ANDROID_HOME 未设置"
    exit 1
else
    echo "ANDROID_HOME=$ANDROID_HOME"
fi

echo "检查项目文件..."
if [ ! -f "package.json" ]; then
    echo "[错误] package.json 不存在"
    exit 1
fi

if [ ! -f "android/local.properties" ]; then
    echo "[警告] android/local.properties 不存在，需要手动创建"
fi

echo ""
echo "[成功] 环境检查完成！"
```

## 📝 部署步骤总结

1. **复制项目** → 将整个项目文件夹复制到新位置
2. **安装依赖** → 运行 `npm install`
3. **配置 SDK** → 创建并编辑 `android/local.properties`
4. **验证环境** → 运行验证脚本或手动检查
5. **启动项目** → 运行 `npm start` 和 `npm run android`

## ⚠️ 注意事项

1. **不要复制以下目录/文件**:
   - `node_modules/` (运行 `npm install` 重新生成)
   - `android/build/` (构建时自动生成)
   - `android/app/build/` (构建时自动生成)
   - `android/.gradle/` (Gradle 缓存)
   - `ios/Pods/` (运行 `pod install` 重新生成)

2. **必须保留的文件**:
   - 所有源代码文件 (`src/`, `android/app/src/`)
   - 所有配置文件 (`package.json`, `tsconfig.json`, 等)
   - `android/gradle/` 目录 (Gradle Wrapper)

3. **需要重新配置的**:
   - `android/local.properties` (每个环境不同)
   - 环境变量 (每个电脑不同)
