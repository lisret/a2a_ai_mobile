# 编译错误修复说明

## 问题描述

在重构原生代码后，编译时出现 `Unresolved reference: BuildConfig` 错误。

## 已修复的问题

### 1. BuildConfig 导入问题

**问题**: `MainApplication.kt` 移动到 `core/` 目录后，包名从 `com.awesomeproject` 变为 `com.awesomeproject.core`，无法直接访问 `BuildConfig`。

**修复**: 在 `MainApplication.kt` 中添加了显式导入：
```kotlin
import com.awesomeproject.BuildConfig
```

## 如果仍然遇到编译错误

### 方法 1: 清理并重新构建

```bash
cd AwesomeProject/android
./gradlew clean
./gradlew assembleRelease
```

### 方法 2: 在 Android Studio 中同步

1. 打开 Android Studio
2. 点击 `File` → `Sync Project with Gradle Files`
3. 等待同步完成
4. 重新构建项目

### 方法 3: 清理所有缓存

```bash
cd AwesomeProject/android
./gradlew clean
./gradlew cleanBuildCache
rm -rf .gradle
rm -rf app/build
./gradlew assembleRelease
```

### 方法 4: 检查 BuildConfig 生成

确保 `build.gradle` 中的配置正确：

```gradle
android {
    namespace "com.awesomeproject"
    defaultConfig {
        applicationId "com.awesomeproject"
        // ...
    }
}
```

`BuildConfig` 类会在构建时自动生成在 `com.awesomeproject` 包下。

## 验证修复

编译成功后，`BuildConfig` 类应该位于：
```
app/build/generated/source/buildConfig/release/com/awesomeproject/BuildConfig.java
```

## 其他可能的问题

### 1. 包名不匹配

确保所有文件的包名与目录结构匹配：
- `core/MainApplication.kt` → `package com.awesomeproject.core`
- `service/AutoGLMAccessibilityService.kt` → `package com.awesomeproject.service`
- 等等...

### 2. 导入路径错误

确保所有导入使用完整包名：
```kotlin
import com.awesomeproject.service.AutoGLMAccessibilityService
import com.awesomeproject.core.MainActivity
```

### 3. AndroidManifest.xml 配置

确保 `AndroidManifest.xml` 中的类名使用正确的路径：
```xml
<application android:name=".core.MainApplication">
    <activity android:name=".core.MainActivity">
    <service android:name=".service.AutoGLMAccessibilityService">
```

## 如果问题仍然存在

1. **查看详细错误日志**:
   ```bash
   ./gradlew assembleRelease --stacktrace --info
   ```

2. **检查所有 Kotlin 文件**:
   确保所有文件都没有语法错误和未解析的引用

3. **检查依赖**:
   确保所有依赖都正确配置在 `build.gradle` 中

4. **重新导入项目**:
   在 Android Studio 中：
   - `File` → `Invalidate Caches / Restart`
   - 选择 `Invalidate and Restart`
