# 编译警告修复说明

## 警告类型

### 1. JavaScript Bundle 警告

这些警告来自 React Native 框架打包后的 bundle 文件，主要是关于未声明的全局变量（如 `setTimeout`, `fetch`, `XMLHttpRequest` 等）。

**原因**：
- React Native 框架在打包时会生成这些警告
- 这些是框架内部的全局变量，在运行时环境中是可用的
- 不影响应用功能

**处理方式**：
- 这些警告是正常的，来自 React Native 框架本身
- 可以通过配置 Gradle 的 `lintOptions` 来抑制显示
- 不影响应用的编译和运行

### 2. Kotlin 编译警告

#### 2.1 变量初始化冗余警告

**位置**：`AccessibilityModule.kt:149`

**问题**：
```kotlin
var service: AutoGLMAccessibilityService? = null
service = AutoGLMAccessibilityService.getInstance()
```

**修复**：

var service = AutoGLMAccessibilityService.getInstance()

#### 2.2 已弃用 API 警告

**位置**：`AccessibilityModule.kt:592`

**问题**：
- `windowManager.defaultDisplay.getRealMetrics()` 在 Android 11+ 已弃用

**修复**：
- 使用 Android 11+ 的新 API：`windowManager.currentWindowMetrics`
- 对于 Android 10 及以下，使用 `@Suppress("DEPRECATION")` 抑制警告

## 修复内容

### 1. Kotlin 代码修复

#### 修复变量初始化冗余
```kotlin
// 修复前
var service: AutoGLMAccessibilityService? = null
service = AutoGLMAccessibilityService.getInstance()

// 修复后
var service = AutoGLMAccessibilityService.getInstance()
```

#### 修复已弃用 API
```kotlin
// 修复后
val result = Arguments.createMap()
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.R) {
    // Android 11+ (API 30+) 使用新 API
    val windowMetrics = windowManager.currentWindowMetrics
    val bounds = windowMetrics.bounds
    result.putInt("width", bounds.width())
    result.putInt("height", bounds.height())
} else {
    // Android 10 及以下使用旧 API（已弃用但兼容）
    @Suppress("DEPRECATION")
    val displayMetrics = android.util.DisplayMetrics()
    @Suppress("DEPRECATION")
    windowManager.defaultDisplay.getRealMetrics(displayMetrics)
    result.putInt("width", displayMetrics.widthPixels)
    result.putInt("height", displayMetrics.heightPixels)
}
```

### 2. Gradle 配置

#### 在 `build.gradle` 中添加
```gradle
// 抑制编译警告
lintOptions {
    quiet true
    abortOnError false
    checkReleaseBuilds false
}

// 抑制 Kotlin 编译警告
kotlinOptions {
    suppressWarnings = true
}
```

## 注意事项

1. **JavaScript Bundle 警告**：
   - 这些警告来自 React Native 框架本身
   - 不影响应用功能
   - 可以通过 `lintOptions` 配置来抑制显示

2. **已弃用 API**：
   - 已使用版本检查，Android 11+ 使用新 API
   - Android 10 及以下使用旧 API（兼容性考虑）

3. **警告抑制**：
   - `lintOptions.quiet = true` 会抑制 lint 警告
   - `kotlinOptions.suppressWarnings = true` 会抑制 Kotlin 编译警告
   - 这些配置不会影响错误提示
