# 任务执行引擎流程抽象图

## 整体架构概览

```
┌─────────────────────────────────────────────────────────────────┐
│                     任务执行引擎架构                              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
        ┌─────────────────────────────────────┐
        │   TaskExecutionEngine (核心引擎)     │
        │   - 任务生命周期管理                 │
        │   - 任务循环控制                     │
        │   - 错误处理和资源清理               │
        └─────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        │                     │                     │
        ▼                     ▼                     ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 模块化组件   │    │ 能力服务层   │    │ 对话服务层   │
│              │    │              │    │              │
│ - 状态管理   │    │ - 无障碍服务 │    │ - API调用    │
│ - 步骤管理   │    │ - ADB服务    │    │ - 解析器     │
│ - 对话历史   │    │ - 悬浮窗服务 │    │ - 提示词     │
│ - 取消机制   │    │ - 截图服务   │    │              │
│ - 错误处理   │    │              │    │              │
└──────────────┘    └──────────────┘    └──────────────┘
```

## 主流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                        任务执行主流程                             │
└─────────────────────────────────────────────────────────────────┘

用户发起任务
    │
    ▼
┌─────────────────────────┐
│ TaskExecutionEngine     │
│ .execute(config)        │
└─────────────────────────┘
    │
    ├─► 初始化阶段
    │   ├─► initialize()
    │   │   ├─► 初始化任务状态模块
    │   │   ├─► 初始化步骤管理模块
    │   │   ├─► 初始化对话历史模块
    │   │   ├─► 初始化取消机制模块
    │   │   └─► 初始化错误处理模块
    │   │
    │   └─► preInitialize()
    │       ├─► 检查无障碍服务
    │       ├─► 初始化能力服务
    │       └─► 检查应用状态
    │
    ├─► 任务循环阶段
    │   │
    │   ┌─────────────────────────────────┐
    │   │ executeTaskLoop()                │
    │   │                                  │
    │   │ while (step < maxSteps) {        │
    │   │                                  │
    │   │   ┌───────────────────────────┐ │
    │   │   │ 1. 检查取消状态            │ │
    │   │   │    if (shouldCancel) break │ │
    │   │   └───────────────────────────┘ │
    │   │           │                     │
    │   │           ▼                     │
    │   │   ┌───────────────────────────┐ │
    │   │   │ 2. 增加步骤计数            │ │
    │   │   │    step++                  │ │
    │   │   └───────────────────────────┘ │
    │   │           │                     │
    │   │           ▼                     │
    │   │   ┌───────────────────────────┐ │
    │   │   │ 3. 执行单个步骤            │ │
    │   │   │    executeStep()          │ │
    │   │   └───────────────────────────┘ │
    │   │           │                     │
    │   │           ▼                     │
    │   │   ┌───────────────────────────┐ │
    │   │   │ 4. 检查完成状态            │ │
    │   │   │    if (isComplete) break  │ │
    │   │   └───────────────────────────┘ │
    │   │           │                     │
    │   │           ▼                     │
    │   │   ┌───────────────────────────┐ │
    │   │   │ 5. 等待UI更新              │ │
    │   │   │    await waitForUIUpdate()│ │
    │   │   └───────────────────────────┘ │
    │   │           │                     │
    │   │           └───► 继续循环        │
    │   │                                  │
    │   └─────────────────────────────────┘
    │
    ├─► 完成阶段
    │   ├─► buildCompletedTask()
    │   ├─► emitTaskComplete()
    │   └─► cleanup()
    │
    └─► 错误处理
        ├─► buildFailedTask()
        ├─► emitTaskFailed()
        └─► cleanup()
```

## 单步骤执行详细流程 (executeStep)

```
┌─────────────────────────────────────────────────────────────────┐
│                     单步骤执行流程 (executeStep)                  │
└─────────────────────────────────────────────────────────────────┘

executeStep(instruction, model)
    │
    ├─► 步骤1: 截图
    │   │
    │   ┌─────────────────────────────┐
    │   │ ScreenshotModule.capture()  │
    │   │                              │
    │   │ 1. 尝试无障碍服务截图        │
    │   │    accessibilityService     │
    │   │    .captureScreen()         │
    │   │                              │
    │   │ 2. 失败则尝试ADB截图         │
    │   │    (如果启用ADB兜底)         │
    │   │    adbService.captureScreen()│
    │   │                              │
    │   │ 3. 验证截图有效性            │
    │   │    (检查是否为1x1占位符)     │
    │   │                              │
    │   └─────────────────────────────┘
    │           │
    │           ▼
    │   ┌─────────────────────────────┐
    │   │ screenshotUri (base64)      │
    │   └─────────────────────────────┘
    │
    ├─► 步骤2: 模型推理
    │   │
    │   ┌─────────────────────────────┐
    │   │ ModelInferenceModule.infer() │
    │   │                              │
    │   │ 1. 选择提示词提供者          │
    │   │    DialogueFactory          │
    │   │    .getPromptProvider(model)│
    │   │                              │
    │   │ 2. 获取系统提示词            │
    │   │    promptProvider           │
    │   │    .getSystemPrompt()        │
    │   │                              │
    │   │ 3. 调用通用API服务           │
    │   │    DialogueService          │
    │   │    .callAPI({               │
    │   │      screenshotUri,         │
    │   │      task: instruction,     │
    │   │      model,                 │
    │   │      systemPrompt,          │
    │   │      conversationHistory,   │
    │   │      onStreamUpdate         │
    │   │    })                       │
    │   │                              │
    │   │    ┌──────────────────────┐ │
    │   │    │ API调用流程           │ │
    │   │    │                      │ │
    │   │    │ 1. 构建请求消息      │ │
    │   │    │    - 系统提示词       │ │
    │   │    │    - 对话历史         │ │
    │   │    │    - 当前截图         │ │
    │   │    │    - 任务指令         │ │
    │   │    │                      │ │
    │   │    │ 2. 发送HTTP请求       │ │
    │   │    │    - 流式/非流式      │ │
    │   │    │                      │ │
    │   │    │ 3. 处理响应           │ │
    │   │    │    - 流式: 累积内容   │ │
    │   │    │    - 非流式: 解析JSON │ │
    │   │    └──────────────────────┘ │
    │   │                              │
    │   │ 4. 选择解析器                │
    │   │    DialogueFactory          │
    │   │    .getParser(model)        │
    │   │                              │
    │   │ 5. 解析响应                  │
    │   │    parser.parseAction(     │
    │   │      apiResponse.content,   │
    │   │      getScreenSize         │
    │   │    )                        │
    │   │                              │
    │   │    ┌──────────────────────┐ │
    │   │    │ 解析流程             │ │
    │   │    │                      │ │
    │   │    │ 1. 提取思考过程      │ │
    │   │    │    (thinking)        │ │
    │   │    │                      │ │
    │   │    │ 2. 输出思考日志      │ │
    │   │    │    console.log()     │ │
    │   │    │                      │ │
    │   │    │ 3. 解析操作指令      │ │
    │   │    │    - type: 操作类型  │ │
    │   │    │    - params: 参数   │ │
    │   │    │                      │ │
    │   │    │ 4. 输出执行日志      │ │
    │   │    │    console.log()     │ │
    │   │    │                      │ │
    │   │    │ 5. 返回TaskAction    │ │
    │   │    └──────────────────────┘ │
    │   │                              │
    │   └─────────────────────────────┘
    │           │
    │           ▼
    │   ┌─────────────────────────────┐
    │   │ TaskAction {                │
    │   │   type: 'click' | 'input' | │
    │   │         'swipe' | 'launch' |│
    │   │         'wait' | ...        │
    │   │   params: {...}             │
    │   │ }                           │
    │   │                             │
    │   │ modelResponse (完整响应)    │
    │   └─────────────────────────────┘
    │
    ├─► 步骤3: 更新对话历史
    │   │
    │   ┌─────────────────────────────┐
    │   │ ConversationHistoryModule    │
    │   │ .addMessage({                │
    │   │   role: 'user',              │
    │   │   content: [                 │
    │   │     { type: 'text', ... },  │
    │   │     { type: 'image_url', ...}│
    │   │   ]                          │
    │   │ })                           │
    │   │                              │
    │   │ .addMessage({                │
    │   │   role: 'assistant',         │
    │   │   content: modelResponse    │
    │   │ })                           │
    │   └─────────────────────────────┘
    │
    ├─► 步骤4: 检查完成状态
    │   │
    │   ┌─────────────────────────────┐
    │   │ 检查 action.type             │
    │   │                              │
    │   │ if (action.type === 'finish')│
    │   │   return { isComplete: true }│
    │   │                              │
    │   │ if (action.type === 'take_over')│
    │   │   // 用户接管，继续执行      │
    │   └─────────────────────────────┘
    │
    ├─► 步骤5: 执行操作
    │   │
    │   ┌─────────────────────────────┐
    │   │ executeAction(action)        │
    │   │                              │
    │   │ switch (action.type) {       │
    │   │                              │
    │   │   case 'click':              │
    │   │     accessibilityService     │
    │   │     .performClickAction(     │
    │   │       action.x,              │
    │   │       action.y               │
    │   │     )                        │
    │   │                              │
    │   │   case 'input':              │
    │   │     accessibilityService     │
    │   │     .performTextInput(      │
    │   │       action.text            │
    │   │     )                        │
    │   │                              │
    │   │   case 'swipe':              │
    │   │     accessibilityService     │
    │   │     .performSwipeAction({    │
    │   │       startX, startY,       │
    │   │       endX, endY,            │
    │   │       duration               │
    │   │     })                       │
    │   │                              │
    │   │   case 'launch':             │
    │   │     handleLaunchAction(      │
    │   │       action.app             │
    │   │     )                        │
    │   │     ┌──────────────────────┐│
    │   │     │ 启动应用流程         ││
    │   │     │                      ││
    │   │     │ 1. 判断包名/应用名   ││
    │   │     │    isPackageName()  ││
    │   │     │                      ││
    │   │     │ 2. 应用名转包名      ││
    │   │     │    appMappingService││
    │   │     │    .getPackageName() ││
    │   │     │                      ││
    │   │     │ 3. 尝试无障碍启动    ││
    │   │     │    accessibility    ││
    │   │     │    .launchApp()     ││
    │   │     │                      ││
    │   │     │ 4. 失败则ADB启动     ││
    │   │     │    (如果启用兜底)    ││
    │   │     │    adbService       ││
    │   │     │    .launchApp()     ││
    │   │     │                      ││
    │   │     │ 5. 都失败则等待手动  ││
    │   │     │    发送事件通知用户  ││
    │   │     └──────────────────────┘│
    │   │                              │
    │   │   case 'back':               │
    │   │     accessibilityService     │
    │   │     .performBack()           │
    │   │                              │
    │   │   case 'home':               │
    │   │     accessibilityService     │
    │   │     .performHome()           │
    │   │                              │
    │   │   case 'wait':               │
    │   │     await sleep(action.duration)│
    │   │                              │
    │   │   case 'take_over':          │
    │   │     handleTakeOverAction()   │
    │   │     // 发送事件通知用户      │
    │   │     // 等待1秒后继续         │
    │   │                              │
    │   │   case 'record_search_box':  │
    │   │     handleRecordSearchBox()  │
    │   │                              │
    │   │   default:                   │
    │   │     throw new Error()        │
    │   │ }                            │
    │   └─────────────────────────────┘
    │
    ├─► 步骤6: 记录步骤
    │   │
    │   ┌─────────────────────────────┐
    │   │ StepManagementModule        │
    │   │ .addStep({                   │
    │   │   step,                     │
    │   │   action,                   │
    │   │   timestamp,                │
    │   │   actionDetails,            │
    │   │   modelResponse             │
    │   │ })                           │
    │   └─────────────────────────────┘
    │
    └─► 步骤7: 等待UI更新
        │
        ┌─────────────────────────────┐
        │ waitForUIUpdate()            │
        │                              │
        │ await sleep(                 │
        │   TASK_CONFIG.UI_UPDATE_DELAY│
        │ )                            │
        │                              │
        │ // 通常为 500-2000ms         │
        └─────────────────────────────┘
```

## 后台执行流程 (Headless JS)

```
┌─────────────────────────────────────────────────────────────────┐
│                   后台任务执行流程 (Headless JS)                  │
└─────────────────────────────────────────────────────────────────┘

应用切换到后台
    │
    ▼
┌─────────────────────────┐
│ TaskExecutionHeadless   │
│ Service 启动             │
└─────────────────────────┘
    │
    ▼
┌─────────────────────────┐
│ registerTaskExecutionTask│
│ (taskDataObj)           │
└─────────────────────────┘
    │
    ├─► 解析任务数据
    │   JSON.parse(taskDataString)
    │
    ├─► 发送任务开始事件
    │   DeviceEventEmitter.emit('TaskStarted')
    │
    ├─► 执行后台任务
    │   executeTaskInBackground()
    │   │
    │   ┌─────────────────────────────┐
    │   │ 1. 初始化后台资源            │
    │   │    - 启动前台服务            │
    │   │    - 获取WakeLock            │
    │   │    - (可选)显示悬浮窗        │
    │   │                              │
    │   │ 2. 进入任务循环              │
    │   │    while (step < maxSteps) { │
    │   │                              │
    │   │      a. 检查取消状态          │
    │   │         DeviceEventEmitter   │
    │   │         .on('TaskCancelRequested')│
    │   │                              │
    │   │      b. 截图                 │
    │   │         (无障碍/ADB)          │
    │   │                              │
    │   │      c. 模型推理              │
    │   │         modelInferenceModule │
    │   │         .infer()             │
    │   │                              │
    │   │      d. 执行操作             │
    │   │         executeAction()      │
    │   │                              │
    │   │      e. 更新前台服务通知      │
    │   │         accessibilityService │
    │   │         .updateTaskExecutionService()│
    │   │                              │
    │   │      f. 等待UI更新            │
    │   │         await sleep()        │
    │   │    }                          │
    │   │                              │
    │   │ 3. 清理资源                  │
    │   │    - 停止前台服务            │
    │   │    - 释放WakeLock            │
    │   │    - 隐藏悬浮窗              │
    │   └─────────────────────────────┘
    │
    ├─► 发送任务完成事件
    │   DeviceEventEmitter.emit('TaskCompleted')
    │
    └─► 发送任务失败事件 (如果失败)
        DeviceEventEmitter.emit('TaskFailed')
```

    │   │    │     role: 'user',    │ │

    │   │    └──────────────────────┘ │
    │   │                              │

        │ )                           │
        │                              │
        │ 1. 提取思考过程              │
        │    ┌──────────────────────┐ │
        │    │ // 从响应中提取       │ │
        │    │ // <thinking>...</thinking>│
        │    │                      │ │
        │    │ thinking = extractThinking()│

        │                              │
        │ 2. 输出思考日志              │
        │    console.log('[思考]', thinking)│
        │                              │
        │ 3. 解析操作指令              │
        │    ┌──────────────────────┐ │
        │    │ // 从响应中提取       │ │
        │    │ // <action>...</action>│
        │    │                      │ │
        │    │ action = parseActionBlock()│
        │    │                      │ │
        │    │ // 转换为TaskAction   │ │
        │    │ // {                 │ │
        │    │ //   type: 'click',  │ │

        │    │ // }                 │ │
        │    └──────────────────────┘ │
        │                              │

        │    console.log('[执行]', action)│
        │                              │
        │ 5. 返回TaskAction            │
        │    return action             │
        └─────────────────────────────┘

## 事件通信机制

```
┌─────────────────────────────────────────────────────────────────┐
│                     事件通信机制                                 │
└─────────────────────────────────────────────────────────────────┘

后台任务 (Headless JS)
    │
    ├─► TaskStarted
    │   { taskId, step: 0, instruction }
    │
    ├─► TaskCompleted
    │   { taskId, step, task }
    │
    ├─► TaskFailed
    │   { taskId, error }
    │
    ├─► TaskManualLaunchRequired
    │   { taskId, step, packageName, message }
    │
    ├─► TaskTakeOverRequired
    │   { taskId, step, message }
    │
    └─► TaskCancelRequested (监听)
        { taskId: 'current' }

前台UI (React Native)
    │
    ├─► 监听 TaskStarted
    │   → 更新UI状态
    │
    ├─► 监听 TaskCompleted
    │   → 显示完成提示
    │   → 更新任务历史
    │
    ├─► 监听 TaskFailed
    │   → 显示错误提示
    │
    ├─► 监听 TaskManualLaunchRequired
    │   → 显示提示，等待用户操作
    │   → 发送 TaskManualLaunchCompleted
    │
    ├─► 监听 TaskTakeOverRequired
    │   → 显示接管提示
    │
    └─► 发送 TaskCancelRequested
        → 取消当前任务
```

## 总结

任务执行引擎采用模块化设计，主要流程包括：

1. **初始化阶段**: 初始化所有模块和资源
2. **任务循环阶段**: 循环执行截图→推理→操作→等待
3. **完成阶段**: 构建任务结果并清理资源

关键特性：
- ✅ 模块化设计，易于扩展
- ✅ 支持后台执行 (Headless JS)
- ✅ 完善的错误处理机制
- ✅ 事件驱动的UI更新
- ✅ 多级降级策略 (无障碍→ADB)
- ✅ 支持流式和非流式API响应
