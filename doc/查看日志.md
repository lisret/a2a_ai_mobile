# 查看 Android 应用日志 (Logcat)

## 方法 1: 使用 ADB 命令（推荐）

### 查看所有日志
```bash
adb logcat
```

### 只查看应用相关日志（过滤）
```bash
# 查看应用包名的日志
adb logcat | grep "com.awesomeproject"

# 查看无障碍服务相关日志
adb logcat | grep -E "(AutoGLMAccessibility|AccessibilityModule)"

# 查看任务执行相关日志
adb logcat | grep "任务执行"

# 查看所有相关日志（无障碍 + 任务执行）
adb logcat | grep -E "(AutoGLMAccessibility|AccessibilityModule|任务执行|无障碍)"
```

### 清除旧日志后查看
```bash
adb logcat -c && adb logcat | grep -E "(AutoGLMAccessibility|AccessibilityModule|任务执行)"
```

### 保存日志到文件
```bash
adb logcat > logcat.txt
# 或者只保存相关日志
adb logcat | grep -E "(AutoGLMAccessibility|AccessibilityModule|任务执行)" > app_log.txt
```

## 方法 2: 使用 Android Studio

1. 打开 Android Studio
2. 连接设备或启动模拟器
3. 点击底部工具栏的 **Logcat** 标签
4. 在过滤框中输入：
   - `AutoGLMAccessibility` - 查看无障碍服务日志
   - `AccessibilityModule` - 查看 React Native 模块日志
   - `任务执行` - 查看任务执行日志
   - `package:com.awesomeproject` - 查看应用所有日志

## 方法 3: 使用 VS Code（如果安装了 Android 扩展）

1. 安装 "Android Logcat" 扩展
2. 在 VS Code 中打开命令面板 (Ctrl+Shift+P)
3. 输入 "Android: Show Logcat"
4. 选择设备并查看日志

## 方法 4: 使用 React Native 调试器

在 Metro bundler 终端中，也会显示部分 JavaScript 日志。

## 常用过滤命令

### 查看无障碍服务检查日志
```bash
adb logcat | grep -E "(AccessibilityModule|无障碍服务)"
```

### 查看截图相关日志
```bash
adb logcat | grep -E "(截图|captureScreen|MediaProjection)"
```

### 查看错误日志
```bash
adb logcat *:E
```

### 查看警告和错误
```bash
adb logcat *:W *:E
```

## 实时监控关键日志

创建一个批处理文件 `watch_logs.bat`（Windows）：
```batch
@echo off
echo 正在监控应用日志...
adb logcat -c
adb logcat | findstr /C:"AutoGLMAccessibility" /C:"AccessibilityModule" /C:"任务执行" /C:"无障碍"
```

或者 PowerShell 脚本 `watch_logs.ps1`：
```powershell
Write-Host "正在监控应用日志..."
adb logcat -c
adb logcat | Select-String -Pattern "AutoGLMAccessibility|AccessibilityModule|任务执行|无障碍"
```

## 调试无障碍服务问题

当遇到"无障碍服务未启用"提示时，运行：
```bash
adb logcat | grep -E "(AccessibilityModule|无障碍服务|isEnabled)"
```

这会显示：
- 检查无障碍服务的详细过程
- 已启用的服务列表
- 服务名称匹配结果
- 最终检查结果

## 注意事项

1. **确保设备已连接**：运行 `adb devices` 确认设备已连接
2. **确保应用已安装**：应用需要在设备上运行
3. **日志级别**：默认显示所有级别，可以使用 `*:V` (Verbose), `*:D` (Debug), `*:I` (Info), `*:W` (Warning), `*:E` (Error) 过滤
