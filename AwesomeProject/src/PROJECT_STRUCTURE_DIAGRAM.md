## 📊 模块职责说明

### Core Layer (核心层)

#### `core/ability/` - 设备能力模块
- **职责**: 提供设备操作能力（截图、点击、滑动等）
- **特点**: 
  - 统一的 `ICapability` 接口
  - 支持多种能力实现（Accessibility、ADB）
  - 通过 `CapabilityService` 统一访问

#### `core/engine/dialogue/` - 对话模块
- **职责**: 模型无关的对话逻辑
- **特点**:
  - 工厂模式选择提示词和解析器
  - 支持多模型扩展
  - 接口驱动设计

#### `core/engine/openAutoGML/` - Open-AutoGLM 引擎
- **职责**: 任务执行的核心逻辑
- **特点**:
  - 模块化设计
  - 职责分离
  - 易于测试和维护

### Features Layer (功能层)

#### `features/model/` - 模型管理
- **职责**: AI模型的管理功能
- **包含**: 列表、添加、编辑、服务

#### `features/task/` - 任务功能
- **职责**: 任务执行和历史管理
- **包含**: 历史查看、执行控制、后台执行

#### `features/settings/` - 设置功能
- **职责**: 应用设置管理
- **包含**: 设置页面、API密钥指南

#### `features/debug/` - 调试功能
- **职责**: 调试日志查看
- **包含**: 日志屏幕、日志服务

### Shared Layer (共享层)

#### `shared/components/` - 通用组件
- **职责**: 可复用的UI组件
- **特点**: 无业务逻辑，纯展示组件

#### `shared/constants/` - 常量配置 ⭐
- **职责**: 集中管理所有硬编码配置
- **特点**: 
  - 分类配置文件，便于维护
  - 统一导出入口
  - 所有常量集中管理
- **包含**: 
  - `task.config.ts` - 任务执行配置
  - `api.config.ts` - API配置
  - `ui.config.ts` - UI配置
  - `storage.config.ts` - 存储配置
  - `app.config.ts` - 应用配置
  - `messages.config.ts` - 消息配置

#### `shared/types/` - 类型定义
- **职责**: 共享类型定义
- **包含**: 模型类型、导航类型

#### `shared/utils/` - 工具函数
- **职责**: 通用工具函数
- **包含**: 格式化、存储、验证、任务辅助

## 🔄 依赖关系规则

```
Features → Shared → Core
    ↓         ↓       ↓
   UI层    工具层   引擎层
```

**依赖规则**:
- ✅ Features 可以依赖 Shared 和 Core
- ✅ Shared 可以依赖 Core
- ✅ Core 不依赖 Features 和 Shared（除了类型）
- ❌ 禁止反向依赖

## 🎯 设计原则

1. **单一职责**: 每个模块只负责一个功能
2. **依赖倒置**: 通过接口定义依赖关系
3. **开闭原则**: 对扩展开放，对修改关闭
4. **接口隔离**: 使用接口而非具体实现
5. **模块化**: 按功能域组织代码
6. **配置集中**: 所有硬编码配置统一管理，便于维护和修改

## 📈 扩展点

1. **新模型支持**: 在 `dialogue/` 下添加新的实现
2. **新能力支持**: 在 `ability/` 下添加新的能力实现
3. **新功能模块**: 在 `features/` 下添加新的功能
4. **新组件**: 在 `shared/components/` 下添加通用组件
5. **新增配置**: 在 `shared/constants/` 对应的配置文件中添加

## ⚙️ 配置管理说明

### 配置分类

所有硬编码配置已集中管理到 `shared/constants/` 目录下，按功能分类：

- **task.config.ts** - 任务执行相关配置
  - 最大步数、超时时间、重试次数、等待时间等

- **api.config.ts** - API调用相关配置
  - API提供商、请求参数、超时配置等

- **ui.config.ts** - UI展示相关配置
  - 颜色、尺寸、样式、动画等

- **storage.config.ts** - 存储相关配置
  - AsyncStorage键名、缓存键名等

- **app.config.ts** - 应用功能相关配置
  - ADB配置、无障碍配置、应用映射等

- **messages.config.ts** - 消息文本相关配置
  - 错误提示、UI文本、确认消息等

### 使用方式

```typescript
// ✅ 正确：从配置中导入
import { 
  TASK_CONFIG, 
  API_REQUEST_CONFIG,
  COLORS,
  STORAGE_KEYS 
} from '@shared/constants';

// 使用任务配置
const maxSteps = TASK_CONFIG.MAX_STEPS;

// 使用API配置
const temperature = API_REQUEST_CONFIG.TEMPERATURE;

// 使用UI配置
const primaryColor = COLORS.primary;

// 使用存储键
await AsyncStorage.setItem(STORAGE_KEYS.MODELS, data);

// ❌ 错误：硬编码
const maxSteps = 50;
const temperature = 0.0;
const primaryColor = '#2563EB';
await AsyncStorage.setItem('@autoglm:models', data);
```
