# 无障碍服务启动应用调研报告

## 调研问题
能否通过包名，用无障碍服务打开app？

## 调研结论
**可以，但需要明确实现方式**

无障碍服务（AccessibilityService）本身不提供直接启动应用的方法，但可以通过以下方式实现：

### 方式1：在无障碍服务模块中使用 Intent API（当前实现）

**实现位置**：`AccessibilityActionModule.kt`

**实现原理**：
```kotlin
// 获取应用的启动Intent
val pm = reactApplicationContext.packageManager
val intent = pm.getLaunchIntentForPackage(packageName)
intent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
reactApplicationContext.startActivity(intent)
```

**优点**：
- ✅ 不需要系统权限
- ✅ 适用于所有Android设备
- ✅ 可以检查应用是否已安装
- ✅ 比shell命令更可靠

**缺点**：
- ⚠️ 严格来说不是"无障碍服务直接启动"，而是通过无障碍模块调用Android Intent API
- ⚠️ 需要应用有Context（无障碍服务模块有ReactApplicationContext）

### 方式2：在无障碍服务类中直接启动（可选实现）

**实现位置**：`AutoGLMAccessibilityService.kt`

**实现原理**：
```kotlin
// 在无障碍服务类中添加方法
fun launchApp(packageName: String): Boolean {
    return try {
        val pm = packageManager
        val intent = pm.getLaunchIntentForPackage(packageName)
        intent?.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
        intent?.addFlags(Intent.FLAG_ACTIVITY_CLEAR_TOP)
        startActivity(intent)
        true
    } catch (e: Exception) {
        Log.e(TAG, "启动应用失败", e)
        false
    }
}
```

**优点**：
- ✅ 直接在无障碍服务类中实现，更符合"用无障碍服务启动"的概念
- ✅ 无障碍服务本身有Context，可以直接调用startActivity()

**缺点**：
- ⚠️ 与当前架构略有不同（当前是通过React Native模块调用）

## 当前项目实现情况

### 已实现功能
✅ **已实现通过包名启动应用**

**调用链**：
1. TypeScript层：`AccessibilityService.ts` → `launchApp(appPackage: string)`
2. React Native层：`AccessibilityActionModule.launchApp(packageName)`
3. Android原生层：使用`PackageManager.getLaunchIntentForPackage()` + `startActivity()`

**代码位置**：
- TypeScript接口：`src/abilityModule/code/Accessibility/AccessibilityService.ts:516`
- React Native模块：`android/app/src/main/java/com/awesomeproject/AccessibilityActionModule.kt:167`
- 无障碍服务：`android/app/src/main/java/com/awesomeproject/AutoGLMAccessibilityService.kt`

### 实现细节

**1. 包名验证**
```kotlin
// 检查应用是否已安装
pm.getPackageInfo(packageName, 0)
```

**2. Intent获取**
```kotlin
// 获取应用的主Activity启动Intent
val intent = pm.getLaunchIntentForPackage(packageName)
```

**3. Intent标志**
```kotlin
// 确保在新任务中启动
intent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
intent.addFlags(Intent.FLAG_ACTIVITY_CLEAR_TOP)
```

**4. 错误处理**
- 应用未安装 → 返回错误
- 无法获取Intent → 返回错误（可能应用没有启动Activity）
- 启动失败 → 抛出异常，可回退到ADB方式

**5. 回退机制**
如果无障碍方式失败，可以回退到ADB方式：
- 使用`monkey`命令
- 使用`am start`命令

## 技术要点

### 无障碍服务启动应用的限制

1. **权限要求**
   - ✅ 不需要系统权限
   - ✅ 只需要普通应用权限
   - ✅ 无障碍服务已启用即可

2. **Context要求**
   - ✅ 无障碍服务有Context（继承自Service）
   - ✅ React Native模块有ReactApplicationContext
   - ✅ 两者都可以调用startActivity()

3. **应用要求**
   - ✅ 应用必须已安装
   - ✅ 应用必须有启动Activity（LAUNCHER category）
   - ⚠️ 某些系统应用可能无法启动

### 与ADB方式的对比

| 特性 | 无障碍方式 | ADB方式 |
|------|-----------|---------|
| 权限要求 | 普通权限 | 可能需要系统权限 |
| 可靠性 | 高 | 中等（某些设备受限） |
| 速度 | 快 | 中等（需要shell命令） |
| 适用场景 | 所有已启用无障碍的设备 | 需要USB调试的设备 |
| 错误处理 | 可以检查应用是否安装 | 只能通过命令返回判断 |

## 总结

**答案：可以**

1. ✅ **当前已实现**：通过无障碍模块（AccessibilityActionModule）使用Intent API启动应用
2. ✅ **技术可行**：无障碍服务有Context，可以调用startActivity()
