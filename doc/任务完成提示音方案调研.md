# 任务完成提示音方案调研

## 需求分析

在任务完成后，需要播放提示音或语音通报，提醒用户任务已完成。

## 方案对比

### 方案1：铃声/提示音（Ringtone/Notification Sound）

#### 实现方式
- 使用 Android 的 `RingtoneManager` 或 `MediaPlayer`
- 播放系统通知音或自定义音效文件

#### 优点
- ✅ **实现简单**：代码量少，易于维护
- ✅ **无需权限**：使用系统 API，不需要额外权限
- ✅ **性能好**：资源占用小，响应快
- ✅ **兼容性好**：所有 Android 版本都支持
- ✅ **用户体验**：通知音可以明确表示任务完成

#### 缺点
- ⚠️ **内容固定**：只能播放固定的音效，不能自定义文本内容
- ⚠️ **信息量少**：用户需要查看通知才能知道具体信息

#### 权限要求
- ✅ **无需权限**：使用系统 API，不需要任何权限

#### 实现复杂度
- ⭐⭐☆☆☆ (简单)

---

### 方案2：语音通报（Text-to-Speech, TTS）

#### 实现方式
- 使用 Android 的 `TextToSpeech` API
- 将文本转换为语音播放

#### 优点
- ✅ **信息明确**：可以播放自定义文本（如"任务完成"）
- ✅ **用户体验好**：用户无需查看通知即可知道任务状态

#### 缺点
- ⚠️ **需要初始化**：需要等待 TTS 引擎初始化（可能需要几秒）
- ⚠️ **可能需权限**：某些设备可能需要 TTS 数据下载权限
- ⚠️ **实现复杂**：需要处理 TTS 引擎状态、语言设置等
- ⚠️ **性能开销**：TTS 引擎占用资源较大
- ⚠️ **兼容性问题**：某些设备可能没有 TTS 引擎或需要下载

#### 权限要求
- ⚠️ **可能需要权限**：
  - 某些设备需要 `INTERNET` 权限（下载 TTS 数据）
  - 某些设备需要用户手动安装 TTS 引擎

#### 实现复杂度
- ⭐⭐⭐⭐☆ (较复杂)

---

## 方案选择

### 推荐方案：**铃声/提示音（方案1）**

**理由：**
1. ✅ **实现简单**：代码量少，易于维护
2. ✅ **无需权限**：符合用户要求（权限少）
3. ✅ **性能好**：响应快，资源占用小
4. ✅ **兼容性好**：所有 Android 版本都支持
5. ✅ **用户体验足够**：通知音可以明确表示任务完成

### 备选方案：**语音通报（方案2）**

如果用户需要更详细的信息（如"任务完成，共执行5个步骤"），可以考虑使用 TTS，但需要：
- 处理 TTS 引擎初始化
- 处理可能的权限问题
- 处理 TTS 引擎不可用的情况

## 实现方案（推荐：铃声）

### 技术实现

#### 1. 使用 RingtoneManager（推荐）
```kotlin
// 播放系统默认通知音
val notification = RingtoneManager.getDefaultUri(RingtoneManager.TYPE_NOTIFICATION)
val ringtone = RingtoneManager.getRingtone(context, notification)
ringtone?.play()
```

**优点：**
- 最简单
- 使用系统默认通知音，用户熟悉
- 无需权限

#### 2. 使用 MediaPlayer（备选）
```kotlin
// 播放自定义音效文件
val mediaPlayer = MediaPlayer.create(context, R.raw.task_complete_sound)
mediaPlayer?.start()
```

**优点：**
- 可以播放自定义音效
- 可以控制音量、循环等

**缺点：**
- 需要将音效文件打包到应用中
- 增加应用体积

### 推荐实现：使用 RingtoneManager

**原因：**
- 最简单，无需额外资源
- 使用系统通知音，用户熟悉
- 无需权限

## 最终方案

### 选择：**使用 RingtoneManager 播放系统通知音** ✅

**实现步骤：**
1. ✅ 在原生代码中添加播放提示音方法（`SoundManager.kt`）
2. ✅ 在 AccessibilityModule 中暴露给 React Native
3. ✅ 在 AccessibilityService 中添加 TypeScript 接口
4. ✅ 在 SettingsService 中添加开关
5. ✅ 在设置页面添加开关 UI
6. ✅ 在任务完成时检查开关并播放提示音

**权限要求：**
- ✅ **无需权限**：使用系统 API，不需要任何额外权限

**用户体验：**
- 任务完成时播放系统通知音
- 用户可以通过设置开关控制是否播放
- 通知音与系统通知音一致，用户熟悉
- 在显示提示窗前播放，确保用户能及时收到提醒

## 实现细节

### 1. 原生代码实现

**文件：** `SoundManager.kt`
```kotlin
class SoundManager(private val context: Context) {
    fun playTaskCompletionSound(): Boolean {
        // 使用 RingtoneManager 播放系统默认通知音
        val notificationUri = RingtoneManager.getDefaultUri(RingtoneManager.TYPE_NOTIFICATION)
        val ringtone = RingtoneManager.getRingtone(context, notificationUri)
        ringtone?.play()
        return true
    }
}
```

**特点：**
- 使用系统默认通知音，无需额外资源
- 无需权限
- 实现简单，代码量少

### 2. React Native 桥接

**文件：** `AccessibilityModule.kt`
```kotlin
@ReactMethod
fun playTaskCompletionSound(promise: Promise) {
    val success = soundManager.playTaskCompletionSound()
    promise.resolve(success)
}
```

### 3. TypeScript 接口

**文件：** `AccessibilityService.ts`
```typescript
async playTaskCompletionSound(): Promise<void> {
    await AccessibilityModule.playTaskCompletionSound();
}
```

### 4. 设置服务

**文件：** `SettingsService.ts`
```typescript
async getTaskCompletionSoundEnabled(): Promise<boolean>
async setTaskCompletionSoundEnabled(enabled: boolean): Promise<void>
```

**默认值：** `true`（默认启用）

### 5. 设置页面 UI

**位置：** `SettingsScreen.tsx`
- 添加了"任务完成提示音"开关
- 位于"应用设置"分组中
- 与"ADB 兜底运行"开关并列

### 6. 任务完成时调用

**文件：** `TaskExecutionHeadless.ts`

**任务完成时：**
```typescript
// 在显示提示窗前播放提示音
const soundEnabled = await settingsService.getTaskCompletionSoundEnabled();
if (soundEnabled) {
  await accessibilityService.playTaskCompletionSound();
}
```

**任务失败时：**
```typescript
// 同样在显示提示窗前播放提示音
const soundEnabled = await settingsService.getTaskCompletionSoundEnabled();
if (soundEnabled) {
  await accessibilityService.playTaskCompletionSound();
}
```

## 执行流程

```
任务完成/失败
    │
    ├─► 1. 保存任务记录（阻塞）
    │
    ├─► 2. 更新前台服务通知（阻塞）
    │
    ├─► 3. 发送事件到前台（非阻塞）
    │
    ├─► 4. 播放提示音（阻塞，如果启用）
    │   └─► 检查设置开关
    │       ├─► 启用：播放系统通知音
    │       └─► 禁用：跳过播放
    │
    └─► 5. 显示提示窗（阻塞）
        ├─► 系统对话框
        └─► 通知栏通知
```
