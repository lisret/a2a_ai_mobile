## ⭐ 关键文件说明

### 必须配置的文件

1. **android/local.properties** ⚠️
   - 必须手动创建
   - 配置 Android SDK 路径
   - 不同电脑路径不同

2. **package.json**
   - 包含所有依赖和脚本
   - 运行 `npm install` 自动安装依赖

3. **tsconfig.json**
   - TypeScript 配置
   - 路径别名配置

4. **babel.config.js**
   - Babel 转译配置
   - 模块路径解析配置

### 重要配置文件

1. **AndroidManifest.xml**
   - Android 应用清单
   - 权限声明
   - 服务注册

2. **android/app/build.gradle**
   - Android 应用构建配置
   - 依赖管理
   - 签名配置

3. **android/build.gradle**
   - 项目级构建配置
   - Gradle 版本
   - 仓库配置

## 🔍 文件查找指南

### 需要修改 Android 原生功能？

→ 查看 `android/app/src/main/java/com/awesomeproject/`

### 需要修改 React Native 代码？

→ 查看 `src/` 目录

### 需要修改应用配置？

→ 查看 `app.json`、`AndroidManifest.xml`、`Info.plist`

### 需要添加新的依赖？

→ 修改 `package.json`，然后运行 `npm install`

### 需要修改构建配置？

→ 查看 `android/build.gradle` 和 `android/app/build.gradle`

## 📋 部署检查清单

在新环境中部署时，确保以下文件/目录存在：

- [ ] `package.json` - 项目依赖配置
- [ ] `android/local.properties` - Android SDK 路径 (需创建)
- [ ] `android/app/src/main/AndroidManifest.xml` - Android 清单
- [ ] `android/app/src/main/java/` - Kotlin 源代码
- [ ] `src/` - React Native 源代码
- [ ] `tsconfig.json` - TypeScript 配置
- [ ] `babel.config.js` - Babel 配置

## 🚨 注意事项

1. **不要提交到 Git 的文件**:
   - `node_modules/`
   - `android/build/`
   - `android/app/build/`
   - `android/local.properties`
   - `ios/Pods/`
   - `*.log`

2. **需要手动创建的文件**:
   - `android/local.properties` (每个环境不同)

3. **敏感信息**:
   - `android/app/debug.keystore` (调试密钥)
   - 生产环境密钥文件 (不应提交到 Git)
